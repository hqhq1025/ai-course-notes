\section{引言}
本期介绍 veRL 中简洁高效的 FSDP SFT Trainer，回顾 SFT 训练的完整过程。

\section{好的训练框架设计}
\begin{importantbox}{两大设计维度}
\begin{itemize}
  \item \textbf{Dataset}：数据加载和预处理抽象（SFT Dataset, RLHF Dataset）
  \item \textbf{Trainer}：训练循环抽象（SFT Trainer, PPO Trainer）
\end{itemize}
\end{importantbox}

\section{SFT Dataset}
核心是 \texttt{get\_item} 函数：将 prompt-response 数据转换为模型输入。关键点：
\begin{itemize}
  \item 只对 response 部分计算 loss（prompt 部分 mask 掉）
  \item Token 化处理：添加特殊 token、padding、attention mask
\end{itemize}

\section{FSDP SFT Trainer}
veRL 的 SFT Trainer 代码极其简洁清晰，是学习 SFT 实现细节的优秀模板。FSDP（Fully Sharded Data Parallel）实现了分布式训练。

\section{SFT vs. RL 训练差异}
\begin{itemize}
  \item SFT：有监督学习，有明确的 target，不需要 rollout
  \item RL：需要在线 rollout 生成 response，reward 作为训练信号
\end{itemize}

\subsection{本章小结}
veRL 的 SFT Trainer 是一个优秀的参考实现，代码简洁、可读性强。

\section{总结与延伸}
\begin{enumerate}
  \item Dataset + Trainer 是训练框架的两大核心抽象
  \item SFT 只对 response 计算 loss
  \item FSDP 实现分布式训练
\end{enumerate}
