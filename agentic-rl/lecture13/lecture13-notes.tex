\section{引言}
本期深入 veRL 的 Agentic Loop 代码，详细分析 \texttt{agentloop.py} 的实现。

\section{核心类结构}
\subsection{AgentLoopBase}
抽象基类，\texttt{run} 方法是抽象方法，需要用户自定义。官方提供了 Singleton 和 TwoAgents 两种实现。

\subsection{AgentLoopWorker}
封装了单个 Agent 的执行逻辑，管理状态转换。

\subsection{AgentLoopManager}
最核心的类，通过 \texttt{generate\_sequence} 方法协调所有组件：
\begin{itemize}
  \item 管理推理 server
  \item 协调 Worker 的状态
  \item 处理 tool call 的异步执行
\end{itemize}

\section{设计理念}
veRL 的 Agentic Loop 追求：
\begin{itemize}
  \item 简洁的抽象
  \item 高效的异步执行
  \item 灵活的自定义接口
\end{itemize}

\subsection{本章小结}
AgentLoopManager 是 veRL Agentic RL 的大脑，统一管理推理、工具调用和训练的协调。

\section{总结与延伸}
\begin{enumerate}
  \item AgentLoopBase $\to$ Worker $\to$ Manager 的层次结构
  \item generate\_sequence 是核心协调方法
  \item 自定义 Agent 只需实现 AgentLoopBase.run
\end{enumerate}
