\section{引言}
本期回答 Agentic Loop 实现中的关键问题：为什么要异步？如何管理不同状态？

\section{为什么要异步？}
\begin{importantbox}{异步的必要性}
Agentic Loop 中，不同 sample 的 Multi-turn 步数不同。如果同步等待最慢的 sample，效率极低。异步执行让已完成的 sample 先返回，最大化 GPU 利用率。
\end{importantbox}

\section{状态管理}
Agentic Loop 中每个 sample 可能处于不同状态：
\begin{itemize}
  \item \textbf{Generating}：正在生成文本
  \item \textbf{Tool Calling}：正在执行工具
  \item \textbf{Waiting}：等待工具返回
  \item \textbf{Done}：已完成所有轮次
\end{itemize}

状态转换由配置文件控制：配置了 tool 则调用 tool，配置了 interaction 则模拟用户反馈。

\subsection{本章小结}
异步执行和精细的状态管理是 Agentic Loop 高效运行的关键。

\section{总结与延伸}
\begin{enumerate}
  \item 异步执行解决了不同 sample 步数不同的效率问题
  \item 状态管理确保每个 sample 在正确的阶段执行正确的操作
  \item 配置驱动的设计提供了灵活性
\end{enumerate}
