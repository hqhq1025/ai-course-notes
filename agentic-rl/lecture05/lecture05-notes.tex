\section{引言}
本期展开 vLLM 推理和部署的更多参数和细节，包括显存分析和调优方法。vLLM 有两大应用场景：推理部署和 RL Training 中的在线 rollout。

\begin{importantbox}{vLLM 的两大场景}
\begin{itemize}
  \item \textbf{推理部署}：Offline Batch Generation + Online Serving（OpenAI API 兼容）
  \item \textbf{RL Training}：在 RL for Language Model 中做模型的 online rollout（如 OpenRLHF 首次引入现代推理引擎到 RL pipeline）
\end{itemize}
\end{importantbox}

\section{Online vs. Offline 推理}
\subsection{Offline Batch Generation}
适用于批量处理场景，一次性处理大量请求，最大化吞吐量。

\subsection{Online Serving}
部署为 OpenAI API 兼容的服务，支持实时请求。关键指标：TTFT（首 token 延迟）和 TPS（每秒 token 数）。

\section{显存分析}
模型权重 + KV Cache + 中间激活值。KV Cache 随序列长度线性增长，是长上下文推理的主要瓶颈。

\subsection{本章小结}
vLLM 的显存消耗主要由模型权重和 KV Cache 决定。合理的参数配置是高效推理的关键。

\section{总结与延伸}
\begin{enumerate}
  \item vLLM 同时服务于推理部署和 RL Training
  \item Offline 和 Online 两种模式各有优化方向
  \item 显存分析：模型权重 + KV Cache + 激活值
\end{enumerate}
