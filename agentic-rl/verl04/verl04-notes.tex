\section{引言}
本期补充 SFT 的更多细节：SFT 训练时不需要 decoding，以及 Cross-Entropy Loss 的计算。

\begin{importantbox}{SFT 训练不需要 Decoding}
SFT 是有监督学习，我们已经知道 target response。训练时\textbf{不需要}模型逐 token 生成（decoding），而是一次性并行计算所有位置的 loss。这与推理时的 auto-regressive generation 有本质区别。
\end{importantbox}

\section{Cross-Entropy Loss 详解}
对于 GPT 这样的 decode-only 模型：
\begin{itemize}
  \item 输入完整序列（prompt + response）
  \item 模型并行输出所有位置的 logits
  \item 只对 response token 的位置计算 cross-entropy loss
  \item Prompt 部分通过 loss mask 屏蔽
\end{itemize}

\section{Training vs. Inference}
\begin{warningbox}{关键区别}
\begin{itemize}
  \item \textbf{Training}：Teacher Forcing，并行计算所有位置
  \item \textbf{Inference}：Auto-regressive，逐 token 生成
\end{itemize}
\end{warningbox}

\subsection{本章小结}
SFT 的高效性来自 Teacher Forcing：并行计算所有位置的 loss，无需逐 token decoding。

\section{总结与延伸}
\begin{enumerate}
  \item SFT 训练不需要 decoding（Teacher Forcing）
  \item 只对 response 部分计算 Cross-Entropy Loss
  \item 这是 SFT 比 RL 训练快得多的根本原因
\end{enumerate}
