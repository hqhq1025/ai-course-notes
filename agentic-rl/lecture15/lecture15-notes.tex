\section{引言}
本期介绍基于 Docker 的 veRL 安装方式，实现无痛的环境配置。

\begin{knowledgebox}{为什么用 Docker？}
传统的 pip install 方式经常遇到 Flash Attention 和 PyTorch 版本依赖冲突。Docker 提供了隔离的、可复现的环境，避免了这些问题。
\end{knowledgebox}

\section{Docker 基本概念}
\subsection{镜像与容器}
\begin{itemize}
  \item \textbf{Image}（镜像）：只读模板，包含操作系统和软件环境
  \item \textbf{Container}（容器）：镜像的运行实例，可读写
  \item 关系类似于``类和``对象
\end{itemize}

\subsection{宿主机与容器}
开发环境：Mac（本地终端）$\to$ 远程 GPU Server $\to$ Docker 容器中运行 veRL。通过 VSCode/Cursor 连接远程容器，实现本地开发体验。

\section{核心命令}
\texttt{docker run -it --rm -v <host>:<container> -w <workdir> <image>:<tag> <cmd>}
\begin{itemize}
  \item \texttt{-it}：交互式终端
  \item \texttt{--rm}：退出后删除容器
  \item \texttt{-v}：磁盘挂载（宿主机目录映射到容器）
  \item \texttt{-w}：设置工作目录
\end{itemize}

\subsection{本章小结}
Docker 是管理复杂 ML 环境的最佳实践。veRL 官方提供了 Docker 镜像，可快速搭建训练环境。

\section{总结与延伸}
\begin{enumerate}
  \item Docker 解决了环境依赖冲突问题
  \item 镜像 $\to$ 容器 $\to$ 磁盘挂载的工作流
  \item VSCode Remote 连接容器实现本地开发体验
\end{enumerate}
