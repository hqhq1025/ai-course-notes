\section{引言}
本期介绍 veRL 做 RL post-training 时的一个著名 issue：tokenizer 编解码不可逆导致的训练崩溃。

\begin{warningbox}{核心问题}
Tokenizer 的编解码在很多情况下\textbf{不可逆}：
\begin{itemize}
  \item 多个不同的 token ID list 解码后得到相同的文本
  \item 但对同一文本编码只会得到唯一的 token ID list
  \item 即：decode $\to$ encode $\neq$ 原始 token IDs
\end{itemize}
\end{warningbox}

\section{问题的根源}
\subsection{编解码的多对一关系}
对一个文本先 encode 再 decode 是确定的。但对 token IDs 先 decode 再 encode，可能得到不同的 token IDs。这是因为多个 token ID 组合可以解码为相同的文本。

\subsection{对 RL Training 的影响}
在 Multi-turn tool call 场景中：
\begin{enumerate}
  \item 模型生成 token IDs（如 tool call 格式）
  \item 工具返回文本结果
  \item 将结果 encode 回 token IDs 拼接到上下文
  \item 如果 encode 结果与原始 IDs 不同，训练分布出现偏差
  \item 严重时导致训练崩溃
\end{enumerate}

\subsection{以 Qwen2.5-7B 为例}
展示了具体的 token ID 不一致案例，说明问题的普遍性。

\subsection{本章小结}
Tokenizer 编解码不可逆是 Multi-turn RL Training 的隐患，需要在数据处理中特别注意。

\section{总结与延伸}
\begin{enumerate}
  \item Tokenizer encode-decode 不总是可逆的
  \item 这会导致 Multi-turn RL Training 的分布偏差
  \item 解决方案：确保 token IDs 的一致性处理
\end{enumerate}
