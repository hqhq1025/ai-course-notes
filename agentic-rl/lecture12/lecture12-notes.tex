\section{引言}
本期正式进入 Agentic RL 训练的核心：veRL 中的 Agentic Loop 实现。

\begin{importantbox}{Agentic Loop 的概念}
Agent 通过多轮 think-act-observe 循环与环境交互。在 RL 训练中，模型需要在每一步：
\begin{enumerate}
  \item 生成（Generate）：输出文本或 tool call
  \item 调用工具（Tool Call）：执行代码、查询 API 等
  \item 获取反馈（Observe）：拿到工具返回结果
  \item 继续生成：基于反馈继续推理
\end{enumerate}
这个 Multi-turn 过程由配置文件控制。
\end{importantbox}

\section{veRL 的 Agentic Loop 架构}
核心代码文件：\texttt{agentloop.py}，包含：
\begin{itemize}
  \item \texttt{AgentLoopManager}：统一管理所有流程
  \item \texttt{AgentLoopWorker}：执行具体的 Agent 逻辑
  \item \texttt{AgentLoopBase}：抽象基类，需要自定义 \texttt{run} 方法
  \item Server Manager：管理推理服务
\end{itemize}

\section{generate\_sequence 方法}
\texttt{AgentLoopManager} 的 \texttt{generate\_sequence} 方法是核心，将所有组件统一管理起来。

\subsection{本章小结}
veRL 的 Agentic Loop 提供了完整的 Multi-turn RL 训练框架。

\section{总结与延伸}
\begin{enumerate}
  \item Agentic Loop = Multi-turn 的 think-act-observe 循环
  \item veRL 通过 AgentLoopManager 统一管理
  \item 可自定义 Agent 逻辑（SingleTool, TwoAgents 等）
\end{enumerate}
