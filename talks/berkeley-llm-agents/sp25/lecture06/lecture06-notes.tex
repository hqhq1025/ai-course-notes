\section{引言：多模态 AI Agent 的时代}

本讲由 Salesforce 的 Caiming Xiong 主讲，介绍多模态 Agent 从感知到行动的全链路，重点讨论环境基准、数据获取和模型构建。

\begin{importantbox}{为什么需要多模态 Agent}
我们希望 Agent 不仅能阅读文本，还能看、听、说、感知。大多数计算机任务涉及多个应用和界面，需要视觉-语言-动作模型（VLA）来自动化数字交互。
\end{importantbox}

\section{环境与基准测试}

\subsection{OS-World：真实计算机环境}

\begin{importantbox}{OS-World 的核心特性}
\begin{itemize}
    \item 首个\textbf{可扩展的真实计算机环境}，用于多模态 Agent 开发与基准测试
    \item 基于虚拟机的安全隔离环境，支持自由的键鼠控制
    \item 可配置的任务设置：初始状态、后处理、评估脚本
    \item 支持开放式计算机任务，覆盖多种应用
    \item 369个真实世界任务（含跨应用工作流、不可能任务等）
\end{itemize}
\end{importantbox}

Agent 与环境的交互循环：
\begin{enumerate}
    \item 接收用户指令和环境观察（截图、可访问性树、终端输出等）
    \item 生成可执行动作（点击、输入、滚动等，以代码字符串表示）
    \item 虚拟机执行动作，返回新观察
    \item 重复直到任务完成或达到最大步数
\end{enumerate}

\subsection{评估方法}

采用\textbf{基于执行的奖励函数}：每个任务有自定义评估脚本，支持部分完成评分，适应多样化的开放式任务。

\section{数据获取}

\subsection{Agent 数据的稀缺性}

传统方法（人工标注）成本高、耗时长。需要探索更高效的数据获取策略。

\subsection{合成数据与自动化}

通过环境交互自动收集轨迹数据，结合人工审核确保质量。

\section{模型架构}

\subsection{视觉-语言-动作模型}

将视觉编码器与语言模型结合，使 Agent 能基于截图和文本理解生成精确的界面操作。

\begin{warningbox}{当前 Agent 的局限}
即使是最先进的多模态模型（GPT-4V 等），在 OS-World 基准上的成功率仍然很低（个位数百分比），距离实用还有很大距离。
\end{warningbox}

\subsection{本章小结}

多模态 Agent 需要三大支柱：可交互的真实环境、高质量的训练数据、强大的视觉-语言-动作模型。OS-World 提供了统一的开发和评估平台。

\section{总结与延伸}

\subsection{核心信息}

多模态 Agent 是下一代 AI 应用的核心方向。构建可靠的 Agent 需要真实、可交互的评估环境，而非静态基准。当前技术距离自动化日常计算机操作仍有显著差距。

\subsection{拓展阅读}

    \item OS-World：真实计算机环境基准
    \item WebArena / VisualWebArena：Web Agent 评估
    \item SWE-Bench：软件工程 Agent 评估
\end{itemize}
