\section{引言}

本讲由 Meta 的 Jason Weston 主讲，探讨如何训练 LLM 学会推理。

\section{推理的定义与挑战}

\begin{importantbox}{什么是推理}
推理是从已知信息得出新结论的过程。对 LLM 而言，挑战在于区分\textbf{记忆}（直接从训练数据中检索答案）和\textbf{推理}（组合已知事实得出新结论）。
\end{importantbox}

\section{训练推理能力的方法}

\subsection{监督微调（SFT）方法}

在带详细推理步骤的数据上训练：Chain-of-Thought 数据、工具使用轨迹等。

\subsection{强化学习方法}

\begin{importantbox}{RL 驱动的推理训练}
\begin{itemize}
    \item 过程奖励模型（PRM）：对每步推理给予奖励
    \item 结果奖励模型（ORM）：仅对最终答案给予奖励
    \item RLHF / DPO：人类偏好对齐
    \item 可验证奖励 RL（如 DeepSeek-R1 方式）
\end{itemize}
\end{importantbox}

\subsection{推理时扩展}

\begin{knowledgebox}{Test-Time Compute Scaling}
在推理阶段投入更多计算：
\begin{itemize}
    \item Best-of-N 采样 + 验证器
    \item 搜索算法（Beam Search、MCTS）
    \item 自我修正和迭代改进
\end{itemize}
\end{knowledgebox}

\section{开放问题}

\begin{warningbox}{推理能力的上限}
LLM 是否真正在``推理''还是在进行高级模式匹配？分布外泛化能力仍然有限，形式化验证可能是确保推理正确性的唯一可靠手段。
\end{warningbox}

\section{总结与延伸}

训练 LLM 推理需要高质量的推理数据、有效的 RL 训练方法和推理时扩展策略的综合应用。可验证的推理任务（数学、代码）是最有效的训练信号。

\end{itemize}
