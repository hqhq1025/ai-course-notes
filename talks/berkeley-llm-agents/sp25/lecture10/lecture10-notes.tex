\section{引言}

本讲由 Ohio State 的 Yu Su 教授主讲，探讨语言 Agent 的推理、记忆和规划三大核心能力。

\section{推理能力}

\subsection{Chain-of-Thought 及其变体}

\begin{importantbox}{推理范式的演进}
\begin{itemize}
    \item \textbf{Chain-of-Thought (CoT)}：逐步推理
    \item \textbf{ReAct}：推理 + 行动交替进行
    \item \textbf{Tree-of-Thought}：分支探索多条推理路径
    \item \textbf{推理模型}（O1/R1）：内化的长链推理
\end{itemize}
\end{importantbox}

\section{记忆系统}

\begin{knowledgebox}{Agent 的记忆层次}
\begin{itemize}
    \item \textbf{工作记忆}：上下文窗口中的即时信息
    \item \textbf{短期记忆}：当前任务的交互历史
    \item \textbf{长期记忆}：跨任务的经验积累（RAG、向量数据库等）
\end{itemize}
\end{knowledgebox}

\section{规划能力}

\subsection{规划方法}

\begin{itemize}
    \item \textbf{前向规划}：逐步生成动作序列
    \item \textbf{分层规划}：高级目标分解为子目标
    \item \textbf{反思规划}：执行后反思和调整
\end{itemize}

\begin{warningbox}{规划的根本挑战}
LLM 的规划能力本质上受限于自回归生成范式——难以进行全局搜索和回溯。需要外部搜索算法（如 MCTS）来弥补。
\end{warningbox}

\section{总结与延伸}

推理、记忆和规划是构建有效语言 Agent 的三大支柱。当前 Agent 框架通过 Prompt 工程和工具使用来增强这些能力，但根本性的提升可能需要架构层面的创新。

\subsection{拓展阅读}
\begin{itemize}
    \item ReAct / Reflexion / Tree-of-Thought
    \item MemGPT：操作系统启发的 LLM 记忆管理
    \item Voyager：开放式 Agent 学习
\end{itemize}
