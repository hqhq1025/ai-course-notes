\section{引言}

本讲由 Google DeepMind 的 Xinyun Chen 主讲，深入探讨提升 LLM 推理能力的推理时技术。

\section{推理时计算扩展}

\begin{importantbox}{核心思想}
在推理阶段投入更多计算资源可显著提升模型在困难推理任务上的表现，而无需重新训练模型。
\end{importantbox}

\subsection{采样与选择策略}

\begin{enumerate}
    \item \textbf{Best-of-N}：生成 N 个候选答案，用验证器选择最佳
    \item \textbf{多数投票}（Majority Voting / Self-Consistency）：选择出现频率最高的答案
    \item \textbf{加权投票}：结合验证器分数的加权方案
\end{enumerate}

\subsection{搜索方法}

\begin{itemize}
    \item \textbf{Beam Search}：维护 top-K 候选路径
    \item \textbf{Monte Carlo Tree Search (MCTS)}：探索与利用的平衡
    \item \textbf{逐步搜索}：在每个推理步骤进行搜索
\end{itemize}

\section{验证器（Verifier）}

\begin{importantbox}{两类验证器}
\begin{itemize}
    \item \textbf{结果奖励模型（ORM）}：评估最终答案的正确性
    \item \textbf{过程奖励模型（PRM）}：评估每个中间推理步骤的正确性
\end{itemize}
PRM 提供更细粒度的信号，通常效果更好，但训练数据获取更困难。
\end{importantbox}

\section{自我修正与迭代改进}

\begin{knowledgebox}{自我修正的挑战}
直接让 LLM 自我修正可能导致性能下降（正确答案被``修正''为错误）。需要外部反馈信号（如代码执行结果、形式化验证）来引导有效的自我修正。
\end{knowledgebox}

\section{推理时训练与在线适应}

\begin{warningbox}{推理时 RL}
在测试时对特定问题进行短暂的 RL 训练，利用验证信号快速适应。这是 AlphaProof 使用的策略之一。
\end{warningbox}

\section{总结与延伸}

推理时计算扩展是提升 LLM 推理能力的关键方向。有效的验证器、搜索策略和自我修正机制的组合可以在不改变模型权重的情况下显著提升性能。
