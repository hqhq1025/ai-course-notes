\section{引言：企业级 Agent 的现实需求}

Nicolas Chapados 和 Alexandre Lacoste 来自 ServiceNow（全球企业软件公司，25,000+ 员工）。本讲从\textbf{企业实际工作流}的视角审视 AI Agent——不是技术 demo，而是真正帮助企业用户完成日常工作。

\begin{knowledgebox}{企业 Agent 的核心问题}
企业场景中的 Agent 需要回答：用户是谁？他们在做什么工作？他们需要什么样的 AI 辅助？Agent 应该在什么上下文中部署？这些问题决定了 Agent 的设计。
\end{knowledgebox}

\subsection{本章小结}

企业 Agent 的设计必须从用户和工作流出发，而非从技术能力出发。

\section{企业工作流的特点}

\subsection{IT Service Management（ITSM）}

ServiceNow 的核心业务是 IT 服务管理——处理企业内部的技术支持请求：

\begin{itemize}
    \item 员工提交问题（如``我的 VPN 连不上''）
    \item 自动分类、路由到合适的技术团队
    \item 多步骤的诊断和解决流程
    \item 需要访问内部知识库和系统
\end{itemize}

\subsection{企业工作流的挑战}

\begin{warningbox}{企业 Agent 面临的独特挑战}
\begin{itemize}
    \item \textbf{长尾需求}：数千种不同类型的请求，许多是罕见的
    \item \textbf{专有知识}：每个企业的系统和流程都不同
    \item \textbf{合规要求}：操作需要符合企业政策和审计要求
    \item \textbf{多系统集成}：需要与企业现有的数十个系统交互
    \item \textbf{可靠性要求}：企业不能容忍 Agent 犯错导致业务中断
\end{itemize}
\end{warningbox}

\subsection{本章小结}

企业工作流的复杂性、专有性和可靠性要求使得通用 Agent 方案难以直接适用。

\section{TapeAgents：结构化的 Agent 框架}

ServiceNow 开发了 TapeAgents 框架，核心理念是将 Agent 的执行轨迹记录为结构化的``磁带''（tape）：

\begin{importantbox}{TapeAgents 的设计理念}
\begin{itemize}
    \item 所有 Agent 行为（思考、动作、观测）记录在可审计的结构化日志中
    \item 支持重放、调试和优化
    \item 将 Agent 行为分解为可组合的步骤
    \item 天然支持企业所需的合规审计
\end{itemize}
\end{importantbox}

\subsection{本章小结}

TapeAgents 通过结构化的执行轨迹记录，解决了企业 Agent 的可审计性和可调试性需求。

\section{WorkArena \& BrowserGym}

\subsection{WorkArena：企业级 Agent 基准}

\begin{itemize}
    \item 基于 ServiceNow 平台的真实企业任务基准
    \item 包含知识库搜索、表单填写、列表过滤、仪表盘操作等典型企业任务
    \item 提供标准化的评估环境
\end{itemize}

\subsection{BrowserGym：通用浏览器 Agent 环境}

\begin{itemize}
    \item 统一的浏览器交互环境，支持各种 Web Agent 基准
    \item 标准化的 observation space 和 action space
    \item 支持 MiniWoB、WebArena、WorkArena 等多个基准
\end{itemize}

\subsection{本章小结}

WorkArena 和 BrowserGym 为企业级 Agent 研究提供了标准化的评估基础设施。

\section{Web Agent 的研究前沿}

\begin{itemize}
    \item Web Agent 从 2017 年的 MiniWoB 起步，到 2024 年已有大量新框架和基准
    \item 当前最好的 Web Agent 在复杂企业任务上的成功率仍然有限
    \item 关键挑战：长 horizon 任务、动态网页内容、多步骤错误恢复
\end{itemize}

\section{总结与延伸}

\subsection{核心要点}

\begin{enumerate}
    \item 企业 Agent 设计必须以用户和工作流为中心
    \item 可靠性、合规性和可审计性是企业 Agent 的硬性要求
    \item TapeAgents 通过结构化轨迹记录解决审计和调试需求
    \item WorkArena 和 BrowserGym 提供标准化企业 Agent 评估
    \item Web Agent 研究正在快速发展但仍有大量挑战
\end{enumerate}
