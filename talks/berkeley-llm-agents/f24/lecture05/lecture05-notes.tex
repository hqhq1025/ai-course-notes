\section{引言：LLM 在推理与规划中的局限}

Yuandong Tian 是 Meta AI（FAIR）的研究科学家/总监。他的讲座探讨如何融合\textbf{神经方法}（LLM）和\textbf{符号方法}（搜索、规划算法）来构建更强大的决策系统。

\begin{knowledgebox}{LLM 的双面性}
LLM 在内容生成、对话、角色扮演等任务上表现优异，但在需要精确推理和长期规划的任务上仍然不可靠。关键问题：如何利用 LLM 的灵活性同时弥补其推理缺陷？
\end{knowledgebox}

\subsection{本章小结}

纯 LLM 方法在精确推理和复杂规划上存在根本限制，需要与符号方法融合。

\section{LLM 用于规划的问题}

\subsection{规划任务中的失败模式}

\begin{itemize}
    \item LLM 直接生成行动计划时，对约束条件的遵守不可靠
    \item 长 horizon 规划中错误会累积
    \item LLM 缺乏对状态空间的精确追踪能力
\end{itemize}

\begin{warningbox}{LLM 不是可靠的规划器}
LLM 在规划任务中的主要问题：它生成的是``看起来合理''的计划，而非``逻辑上正确''的计划。对于有严格约束的规划问题（如物流、调度），这种不精确性是不可接受的。
\end{warningbox}

\subsection{本章小结}

LLM 的规划能力受限于其概率生成本质——它擅长生成流畅的文本但不擅长精确推理。

\section{神经-符号融合框架}

\subsection{LLM 作为启发式函数}

\begin{importantbox}{融合范式}
不将 LLM 用作端到端的规划器，而是将其作为传统搜索和规划算法中的\textbf{启发式函数}：
\begin{itemize}
    \item LLM 提供候选动作和评估（启发式）
    \item 符号搜索算法（如 MCTS、A*）确保探索的系统性和解的正确性
    \item 结合两者的优势：LLM 的灵活性 + 搜索的精确性
\end{itemize}
\end{importantbox}

\subsection{搜索增强的 LLM}

\begin{itemize}
    \item \textbf{Tree-of-Thoughts}：将 CoT 扩展为树状搜索，LLM 生成多个候选步骤，通过评估选择最优路径
    \item \textbf{MCTS + LLM}：Monte Carlo Tree Search 利用 LLM 作为 rollout policy 和价值函数
    \item \textbf{LLM + 形式化验证}：LLM 生成候选解，符号验证器检查正确性
\end{itemize}

\subsection{代数构造替代梯度下降}

Yuandong Tian 提出了一个有趣的前瞻性思路：未来可能不需要梯度下降——可以通过代数构造（algebraic construction）将小规模解组合成大规模解。

\subsection{本章小结}

神经-符号融合是提升 Agent 决策能力的有效路径：LLM 提供灵活的启发式，符号方法确保精确性和正确性。

\section{实际应用}

\subsection{游戏 AI}

\begin{itemize}
    \item 在棋类游戏中，LLM + MCTS 的组合显著优于纯 LLM
    \item LLM 可以用自然语言理解游戏规则，替代手写规则
\end{itemize}

\subsection{代码生成与验证}

\begin{itemize}
    \item LLM 生成代码候选，符号执行验证正确性
    \item 类似于 AlphaCode 的``生成-测试-选择''范式
\end{itemize}

\subsection{本章小结}

神经-符号融合在游戏 AI 和代码生成中已展示出实际价值。

\section{总结与延伸}

\subsection{核心要点}

\begin{enumerate}
    \item LLM 直接用于规划不可靠，需要与符号方法结合
    \item 最佳融合范式：LLM 作为启发式 + 搜索/规划算法作为骨架
    \item 代数构造可能提供超越梯度下降的新路径
    \item 实际应用包括游戏 AI、代码生成与验证
\end{enumerate}
