\section{引言：开放性为何重要}

Percy Liang 是 Stanford 计算机科学教授，HELM 基准和 Together AI 的创始人。他从一个尖锐的观察切入：基础模型的能力飞速提升，但\textbf{开放性和可访问性却在对称地下降}——从完整的论文、代码、数据、权重，退化为仅有 API 接口。

\begin{importantbox}{访问塑造研究}
我们拥有什么样的访问权限，决定了我们能做什么样的研究。当只有 API 时，我们只能做 prompting 研究；当有完整权重时，我们可以做 fine-tuning、机制解释性、安全审计等深层研究。
\end{importantbox}

\subsection{本章小结}

开放性不仅是价值观问题，更直接影响 AI 研究的深度和方向。封闭 API 限制了科学理解和安全研究的可能性。

\section{开源基础模型的价值}

\subsection{科学研究的必要条件}

\begin{itemize}
    \item \textbf{可复现性}：科学研究的基石是可复现性，封闭模型无法满足
    \item \textbf{可解释性研究}：理解 Transformer 内部机制需要完整的模型权重
    \item \textbf{安全研究}：Red teaming、对齐研究都需要深度模型访问
    \item \textbf{公平评估}：统一评估（如 HELM）需要在标准化条件下运行模型
\end{itemize}

\begin{knowledgebox}{HELM：统一的基础模型评估}
Holistic Evaluation of Language Models（HELM）是 Percy Liang 团队构建的大规模评估平台，旨在在统一条件下比较不同模型的能力。这需要深度模型访问才能实现公平、全面的评估。
\end{knowledgebox}

\subsection{创新生态}

\begin{itemize}
    \item 开源模型催生了繁荣的微调和应用生态（Llama 系列的社区贡献）
    \item 小规模研究者和创业公司可以在开源模型基础上创新
    \item 去中心化的计算资源池化可以实现超出个体能力的研究
\end{itemize}

\subsection{本章小结}

开源基础模型是科学研究、安全审计、公平评估和创新生态的必要条件。

\section{Together AI 与去中心化计算}

Percy Liang 联合创立的 Together AI 致力于降低 AI 研究和部署的门槛：

\begin{itemize}
    \item 提供开源模型的高效推理和微调服务
    \item 探索去中心化计算——让多方贡献算力，协同完成大规模训练
    \item 在小规模实验中保持``scale mindset''——虽然资源有限，但设计时考虑可扩展性
\end{itemize}

\subsection{本章小结}

去中心化计算和高效推理是让更多人参与 AI 研究的关键基础设施。

\section{AI Agent 中的开源挑战}

\begin{warningbox}{Agent 系统的额外复杂性}
Agent 系统不仅涉及模型本身，还涉及工具、环境、编排框架。即使模型开源，如果工具接口、评估环境或编排逻辑不开放，研究者仍然无法复现和改进 Agent 系统。
\end{warningbox}

\begin{itemize}
    \item Agent 评估需要标准化的环境和基准
    \item 多模型组合系统的开源需要新的方法论
    \item 算力挑战在 Agent 系统中更加突出（长周期、多轮交互）
\end{itemize}

\subsection{本章小结}

Agent 系统的开源不仅需要开放模型权重，还需要开放工具、环境和评估基础设施。

\section{总结与延伸}

\subsection{核心要点}

\begin{enumerate}
    \item 基础模型的开放性正在下降，这限制了科学研究和安全审计
    \item 开源模型是可复现性、可解释性研究和公平评估的必要条件
    \item 去中心化计算可以让更多人参与大规模 AI 研究
    \item Agent 系统的开源需要覆盖模型、工具、环境和评估全栈
\end{enumerate}
