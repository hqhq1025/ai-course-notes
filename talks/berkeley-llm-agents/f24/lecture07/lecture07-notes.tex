\section{引言：软件开发 Agent}

Graham Neubig 是 CMU 教授，同时是 All Hands AI 的首席科学家，维护着开源编程 Agent 框架 \textbf{OpenHands}（原 OpenDevin）。本讲系统介绍了 AI Agent 在软件开发中的应用。

\begin{knowledgebox}{为什么软件开发是 Agent 的理想应用场景}
软件开发天然具备 Agent 成功所需的条件：明确的反馈信号（测试通过/失败）、丰富的工具生态（编辑器、终端、浏览器、版本控制）、大量公开训练数据（GitHub）、以及明确的成功标准。
\end{knowledgebox}

\subsection{本章小结}

软件开发 Agent 是 LLM Agent 最成熟的应用方向之一，因为它具备完善的反馈机制和工具生态。

\section{软件开发 Agent 的架构}

\subsection{核心组件}

\begin{importantbox}{软件开发 Agent 的四大能力}
\begin{enumerate}
    \item \textbf{代码理解}：浏览和理解大型代码库（数千文件）
    \item \textbf{代码生成}：编写新代码或修改现有代码
    \item \textbf{测试与调试}：运行测试、分析错误、迭代修复
    \item \textbf{环境交互}：使用终端、浏览器、文件系统等开发工具
\end{enumerate}
\end{importantbox}

\subsection{OpenHands 架构}

OpenHands 提供完整的软件开发 Agent 平台：
\begin{itemize}
    \item 沙箱化的开发环境（Docker 容器）
    \item Agent 可以访问终端、代码编辑器、浏览器
    \item 支持多种 LLM 后端
    \item 支持 ReAct 风格的推理-行动循环
\end{itemize}

\subsection{本章小结}

软件开发 Agent 需要综合代码理解、生成、测试和环境交互四大能力，OpenHands 提供了统一的平台。

\section{SWE-bench：软件工程基准}

\subsection{任务定义}

\begin{itemize}
    \item 给定一个 GitHub 仓库和一个 issue（bug report 或 feature request）
    \item Agent 需要理解代码库、定位相关文件、编写修复补丁
    \item 通过仓库中的测试用例验证补丁正确性
\end{itemize}

\subsection{当前表现}

\begin{itemize}
    \item SWE-bench Lite（300 个较简单的问题）：最好的 Agent 约 40--50\% 解决率
    \item SWE-bench Full（2,294 个问题）：解决率更低
    \item 人类软件工程师作为对照组的表现也不是 100\%
\end{itemize}

\begin{warningbox}{SWE-bench 的局限}
SWE-bench 只覆盖了 bug fix 和 feature implementation，不涉及软件设计、需求分析、代码审查、部署运维等软件工程的其他环节。完整的软件开发自动化还有很长的路要走。
\end{warningbox}

\subsection{本章小结}

SWE-bench 是当前最重要的软件开发 Agent 基准，但只覆盖了软件工程的一小部分。

\section{关键技术挑战}

\subsection{代码库导航}

\begin{itemize}
    \item 真实项目有数千个文件，Agent 需要高效定位相关代码
    \item 搜索策略：关键词搜索、语义搜索、调用图分析
    \item 上下文窗口限制：无法将整个代码库放入 LLM
\end{itemize}

\subsection{长 Horizon 规划}

\begin{itemize}
    \item 复杂修复可能需要数十步操作
    \item 错误会累积，需要及时发现和纠正
    \item 需要在探索和利用之间平衡
\end{itemize}

\subsection{测试生成}

\begin{itemize}
    \item Agent 不仅要写代码，还要写测试来验证自己的修改
    \item 高质量的测试生成仍然是开放问题
\end{itemize}

\subsection{本章小结}

代码库导航、长 horizon 规划和测试生成是当前软件开发 Agent 的三大技术瓶颈。

\section{总结与延伸}

\subsection{核心要点}

\begin{enumerate}
    \item 软件开发是 LLM Agent 最成熟的应用方向之一
    \item OpenHands 提供了完整的开源软件开发 Agent 平台
    \item SWE-bench 是标准评测基准，当前最佳约 40--50\% 解决率
    \item 代码库导航、长 horizon 规划和测试生成是核心挑战
    \item 完整的软件开发自动化还需要覆盖设计、审查、部署等环节
\end{enumerate}
