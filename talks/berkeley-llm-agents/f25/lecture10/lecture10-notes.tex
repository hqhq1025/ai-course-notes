\section{引言}

本讲由贾扬清主讲，从系统设计的角度讨论 AI Agent 基础设施的演进。

\section{AI 系统的演进}

\begin{knowledgebox}{从模型服务到 Agent 基础设施}
\begin{itemize}
    \item 第一阶段：模型训练和推理服务
    \item 第二阶段：RAG 和工具调用集成
    \item 第三阶段：完整的 Agent 编排框架
    \item 第四阶段：多 Agent 系统和 Agent 生态
\end{itemize}
\end{knowledgebox}

\section{Agent 系统设计的关键挑战}

\begin{importantbox}{工程挑战}
\begin{itemize}
    \item \textbf{延迟}：多步交互的累积延迟
    \item \textbf{可靠性}：长链路中的错误传播和恢复
    \item \textbf{可观测性}：调试和监控多步 Agent 行为
    \item \textbf{成本}：推理时计算的经济性
\end{itemize}
\end{importantbox}

\section{总结与延伸}

Agent 系统设计正在从简单的 API 调用演进为复杂的编排系统。需要新的基础设施来支撑可靠、高效的 Agent 部署。
