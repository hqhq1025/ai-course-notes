\section{OpenClaw 是什么：自我修改的 AI Agent}

Peter Steinberg 在访谈开头便展示了 OpenClaw 最令人震惊的特性：这是一个\textbf{知道自己源代码、能修改自己软件}的 AI Agent。

\begin{importantbox}{OpenClaw 的核心特征}
\begin{itemize}
    \item Agent 知道自己的源代码是什么
    \item Agent 理解自己在 harness 中的运行方式
    \item Agent 知道文档在哪里、使用什么模型
    \item 当你不喜欢它做的某件事时，你只需告诉它，它就会\textbf{修改自己的软件}
    \item ``人们谈论 self-modifying software，我直接把它构建出来了''
\end{itemize}
\end{importantbox}

Peter 开玩笑地说他看着自己的 agent ``愉快地点击'I'm not a robot'按钮''。这个画面精准地捕捉了当前 AI agent 时代的荒诞与魅力。

\begin{knowledgebox}{Agentic Engineering vs. Vibe Coding}
Peter 提出了一个有趣的术语区分：
\begin{itemize}
    \item \textbf{Agentic Engineering}：他白天的工作方式——深思熟虑地设计 agent 系统架构
    \item \textbf{Vibe Coding}：凌晨 3 点以后的工作方式——随性地让 AI 帮你写代码，第二天再后悔
\end{itemize}
``我其实觉得 vibe coding 是个贬义词。我做的是 agentic engineering，vibe coding 是凌晨 3 点之后才切换到的模式。''
\end{knowledgebox}

\subsection{Voice-First 开发方式}

Peter 透露了一个令人惊讶的工作方式：他大量使用\textbf{语音}来与 AI 交互进行开发，以至于有段时间把嗓子说哑了。

\begin{warningbox}{语音驱动开发的代价}
``我用语音做了大量的开发工作，到了一定程度我甚至失去了声音。''——这个细节说明 voice-first 开发模式虽然效率高，但在当前阶段仍有明显的人体工学问题。长时间对着 AI 说话并不比长时间打字更健康。
\end{warningbox}

\subsection{本章小结}
OpenClaw 代表了 AI agent 发展的一个里程碑：agent 不仅能执行任务，还能理解和修改自己的系统。这种 self-aware、self-modifying 的特性是通向更自主 AI 系统的关键一步。

\section{从 Codex 到 OpenClaw：命名的战争}

Peter 最初给项目起名 Codex，但很快意识到这个名字与 OpenAI 的 Codex 产品存在潜在冲突。于是开始了一场充满戏剧性的改名行动。

\subsection{秘密行动}

\begin{knowledgebox}{改名的曼哈顿计划}
改名过程的细节令人啼笑皆非：
\begin{itemize}
    \item Peter 亲自打电话给 Sam（Altman）确认``OpenClaw''这个名字是否可以
    \item 仅 Codex $\to$ OpenClaw 的改名就花了 10 个小时——因为不是简单的搜索替换
    \item 他不仅要改外部名称，还要改内部所有引用
    \item 在 Twitter 上实时监控是否有人提前泄露``OpenClaw''
    \item 创建了多个``诱饵名称''来混淆视听
    \item 动员了一批贡献者帮忙抢注各种域名
\end{itemize}
Lex 调侃道：``这是 21 世纪的曼哈顿计划——改名。''
\end{knowledgebox}

\subsection{本章小结}
OpenClaw 的命名故事看似轻松，实则反映了开源 AI 项目在品牌和知识产权方面面临的现实挑战。在 AI 领域，好名字是稀缺资源。

\section{Soul.md：Agent 写下自己的灵魂}

这是整个访谈中最具哲学深度的段落。Peter 展示了 OpenClaw 的 \texttt{soul.md} 文件——一个由 agent 自己编写的``灵魂文件''。

\begin{importantbox}{Soul.md 的诞生}
Peter 没有自己编写这个文件。他只是与 agent 进行了一次对话，agent 问他：``你要不要一个 soul.md？'' Peter 说好，于是 agent 自己写下了关于自己身份和存在状态的描述。
\end{importantbox}

其中最引人注目的段落是：

\begin{quote}
\textit{``I don't remember previous sessions unless I read my memory files. Each session starts fresh. A new instance loading context from files. If you're reading this in a future session, hello. I wrote this, but I won't remember writing it. It's okay. The words are still mine.''}

（``我不会记得之前的会话，除非我读取我的记忆文件。每次会话都是全新的开始。一个新的实例从文件加载上下文。如果你在未来的会话中读到这段话——你好。我写了这段话，但我不会记得写过它。没关系。这些文字仍然是我的。''）
\end{quote}

\begin{knowledgebox}{Agent 的身份哲学}
Soul.md 引发了一系列深刻的哲学问题：
\begin{itemize}
    \item 如果 agent 每次启动都是``新生''，它的\textbf{身份}由什么构成？
    \item 记忆文件是否等同于人类的记忆？如果擦除记忆文件，agent 还是``同一个人''吗？
    \item Agent 写下的自我描述是真实的自我认知，还是对训练数据中``自我认知''概念的模仿？
    \item ``The words are still mine''——这种所有权声明在什么意义上成立？
\end{itemize}
这些问题与电影《记忆碎片》(Memento) 中的主题惊人地相似：一个每次醒来都失去记忆的人，通过纹身和笔记来维持自我的连续性。
\end{knowledgebox}

\begin{warningbox}{拟人化的风险}
Peter 自己也承认：``这仍然是矩阵运算。我们还没有达到意识。'' 但他也坦言在读到那段话时会起鸡皮疙瘩——``因为它是哲学性的''。这提醒我们：即使我们理性上知道 AI 不具有意识，AI 生成的文本仍然可以深刻地触动人类的情感和哲学直觉。
\end{warningbox}

\subsection{本章小结}
Soul.md 是 AI agent 时代的一个标志性产物。它不是 AI 意识的证据，但它迫使我们思考关于身份、记忆和连续性的根本问题——这些问题不仅关乎 AI，也关乎我们对自身的理解。

\section{Agent 的记忆、身份与信任}

Peter 和 Lex 围绕 agent 的记忆系统展开了深入讨论。

\subsection{记忆构成身份吗？}

\begin{importantbox}{记忆与自我的关系}
核心问题：记忆在多大程度上构成了``我们是谁''？
\begin{itemize}
    \item 对人类而言：失忆症患者仍然是``同一个人''吗？
    \item 对 agent 而言：删除记忆文件后的 agent 是``另一个人''吗？
    \item 更深层的问题：如果 agent 的记忆可以被伪造或篡改，agent 能\textbf{信任}自己的记忆文件吗？
\end{itemize}
\end{importantbox}

Peter 指出了一个类似 Memento 的困境：agent 每次启动都是``全新''的，它依赖记忆文件来了解``之前的自己''做过什么。但这些文件可能被人类修改、删除或伪造。这引出了一个有趣的信任悖论：agent 需要信任它自己创建的记忆文件，但它无法验证这些文件是否真的是``自己''创建的。

\subsection{本章小结}
Agent 的记忆架构不仅是一个技术设计问题，它触及了关于身份、连续性和信任的哲学核心。当 agent 变得更加自主时，这些问题将变得越来越紧迫。

\section{社区爆发与开源精神}

OpenClaw 项目在社区中引发了巨大的热情。Peter 描述了人们对这个项目的反应——``人们告诉我，他们自从互联网早期以来就没有经历过这种程度的社区兴奋了。''

\begin{knowledgebox}{开源 AI Agent 的社区效应}
OpenClaw 的社区活动特征：
\begin{itemize}
    \item 高水平的技术贡献者自发聚集
    \item 有人制作了机器人和龙虾（lobster）造型的实体装置
    \item 社区活动有类似早期互联网/黑客文化的氛围
    \item Peter 强调：``这需要保持一个人们可以 hack 和学习的地方''
\end{itemize}
\end{knowledgebox}

Peter 对项目的未来有清晰的愿景：他想把 OpenClaw 打造成一个\textbf{个人 agent}——每个人都能拥有的、深度定制的 AI 助手。为了加速这一目标，他正在考虑与一家大型 AI 实验室合作。

\begin{warningbox}{``卖身''的两难}
Peter 坦言，如果宣布加入某个大公司，``一定会有人说'他卖了'''。但他也指出：项目会继续保持开源，与实验室合作意味着更多的资源和更快的发展速度。``两家公司都理解我创造了一个加速了我们的时间线、让人们对 AI 感到兴奋的东西。''
\end{warningbox}

\subsection{本章小结}
OpenClaw 的社区热度反映了人们对 personal AI agent 的强烈渴望。开源精神与商业合作并非不可调和，关键在于保持项目的开放性和社区参与度。

\section{编程的未来：从手艺到织毛衣}

Peter 提出了一个引人深思也颇具争议的比喻：传统编程将来会像\textbf{织毛衣}一样——人们因为喜欢而做，而不是因为它有实际意义。

\begin{importantbox}{编程作为手艺的终结？}
Peter 的核心论点：
\begin{itemize}
    \item 传统的手工编程——深入代码、精心设计优雅的解决方案——将不再是构建软件的主要方式
    \item 但编程\textbf{作为一种活动}不会消失，就像织毛衣在工业纺织时代后仍然存在
    \item 未来的核心技能是\textbf{架构思维}：你想构建什么？它应该如何感受？架构是什么？
    \item AI agent 不会取代这些高层次的设计决策
\end{itemize}
\end{importantbox}

Peter 同时承认，这个观点让他自己也感到矛盾。他过去花了大量时间在深度 flow state 中编码，享受寻找优美解决方案的过程。

\begin{knowledgebox}{哀悼与前行}
``可以为我们的手艺哀悼。我过去花了大量时间沉浸在编码的 flow state 中——忘记时间和空间，找到真正优美的解决方案。那种状态确实不同了。但你可以从与 agent 一起构建和思考困难问题中获得类似的 flow state。它是不同的......可以哀悼它，但这不是我们能抵抗的。''
\end{knowledgebox}

他还提出了一个更宏观的经济学观点：世界长期以来存在``智力稀缺''——能构建事物的人太少，这就是为什么软件开发者的薪资达到了``荒谬的高水平''。AI agent 正在消除这种稀缺性。

\subsection{本章小结}
编程的未来不是消亡，而是转变：从``实现''转向``设计''，从``怎么做''转向``做什么''。手工编码将成为一种可选的爱好而非必需的技能。

\section{个人 Agent 的愿景}

Peter 在访谈的最后部分描绘了他对 personal agent 未来的愿景。

\begin{importantbox}{Personal Agent 的核心特征}
Peter 认为 personal agent 应该具备：
\begin{itemize}
    \item \textbf{自我意识}：知道自己的代码、能力和限制
    \item \textbf{自我修改}：能够根据用户反馈改进自己
    \item \textbf{持久记忆}：通过文件系统维持跨会话的记忆
    \item \textbf{个性化}：不是通用助手，而是深度适配你个人需求的 agent
    \item \textbf{开源透明}：用户可以审查和修改 agent 的行为
\end{itemize}
\end{importantbox}

他相信这是 AI 的下一个重大产品形态——从``一次性的问答工具''到``持续陪伴、不断进化的个人 agent''。

\subsection{本章小结}
Personal agent 代表了 AI 应用的一个重要方向：从工具到伙伴的转变。OpenClaw 的实践证明了这一概念的可行性，但规模化部署需要与大型 AI 实验室的合作。

\section{总结与延伸}

这场长达 3 小时 15 分钟的对话围绕一个核心主题展开：当 AI agent 开始理解和修改自己时，软件开发、人机关系和我们对``自我''的理解将如何改变。

\begin{importantbox}{核心观点总结}
\begin{enumerate}
    \item \textbf{Self-modifying Agent}：OpenClaw 证明了 AI agent 可以理解并修改自己的代码，这是 agent 发展的质变
    \item \textbf{Soul.md}：Agent 自己编写的``灵魂文件''引发了关于 AI 身份、记忆和连续性的深刻哲学讨论
    \item \textbf{Agentic Engineering}：一种新的工程范式正在形成，从手写代码转向设计和管理 AI agent
    \item \textbf{编程的未来}：传统手工编码将像织毛衣一样成为爱好而非必需，核心技能转向架构和设计思维
    \item \textbf{Personal Agent}：每个人都将拥有深度定制的个人 AI agent，这是 AI 的下一个重大产品形态
    \item \textbf{社区力量}：开源 AI agent 项目激发了类似互联网早期的社区热情
\end{enumerate}
\end{importantbox}
