\section{Agent 的核心技术栈}

\subsection{LLM 作为 Agent 的大脑}
\begin{knowledgebox}{Agent 的技术分层}
\begin{itemize}
  \item \textbf{Perception}：理解用户意图和环境状态
  \item \textbf{Planning}：制定行动计划（CoT, ReAct）
  \item \textbf{Action}：执行工具调用和环境交互
  \item \textbf{Memory}：管理上下文和历史信息
\end{itemize}
\end{knowledgebox}

\subsection{Tool Use 与环境交互}
Agent 通过 Function Calling / Tool Use 与外部工具交互，扩展了 LLM 的能力边界。

\subsection{本章小结}
Agent 的核心是 LLM + Tool Use + Memory + Planning 的协同。

\section{Agent 的挑战与展望}

\subsection{当前挑战}
\begin{warningbox}{Agent 面临的关键挑战}
\begin{itemize}
  \item 长期规划的可靠性
  \item 错误恢复和回溯
  \item 多 Agent 协作的复杂度
  \item 安全性和可控性
\end{itemize}
\end{warningbox}

\subsection{本章小结}
Agent 技术正从 demo 走向生产，但仍面临可靠性和安全性的挑战。

\section{总结与延伸}
\begin{enumerate}
  \item 2025 年是 AI Agent 的爆发之年
  \item LLM + Tool Use + Memory + Planning 是核心技术栈
  \item 可靠性和安全性是落地的关键挑战
  \item 多 Agent 协作是重要研究方向
\end{enumerate}
