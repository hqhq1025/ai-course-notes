\section{为什么机器人还不能自主学习}

\subsection{当前机器人学习的瓶颈}

当前大多数机器人学习系统需要大量人类参与：标注数据、设计奖励函数、人工重置环境、监督训练过程。\textbf{自主学习}（Autonomous Learning）的目标是让机器人尽可能减少人类干预，在真实世界中自主探索和学习。

\begin{importantbox}{自主 RL 的定义}
自主强化学习（Autonomous RL）是指机器人能够：
\begin{enumerate}
    \item \textbf{自主设定目标}：不依赖人类指定具体任务
    \item \textbf{自主收集数据}：不需要人类操作或重置环境
    \item \textbf{自主获取奖励}：不依赖人工标注的奖励信号
    \item \textbf{持续改进}：长时间无人监督地运行和学习
\end{enumerate}
\end{importantbox}

\subsection{三大核心挑战}

\begin{enumerate}
    \item \textbf{奖励设计问题}：如何在没有人工定义奖励的情况下让机器人知道什么是好的行为
    \item \textbf{环境重置问题}：RL 通常假设每个 episode 结束后环境会重置到初始状态，但真实世界没有"重置按钮"
    \item \textbf{安全问题}：自主探索可能导致危险行为
\end{enumerate}

\subsection{本章小结}

自主 RL 的核心目标是去除机器人学习对人类的依赖，但面临奖励设计、环境重置和安全性三大挑战。

\section{自动奖励获取}

\subsection{基于目标图像的奖励}

\begin{importantbox}{视觉目标达成奖励}
一种简单有效的方法是用\textbf{目标图像}来定义奖励：
\begin{itemize}
    \item 给定目标图像 $I_g$（期望状态的照片）
    \item 当前观测图像 $I_t$
    \item 奖励 = $-d(f(I_t), f(I_g))$，其中 $f$ 是特征提取器，$d$ 是距离度量
\end{itemize}
用户只需拍一张目标状态的照片，无需编写复杂的奖励函数。
\end{importantbox}

\subsection{基于成功分类器的奖励}

另一种方法是训练一个二分类器来判断当前状态是否"成功"：
\begin{itemize}
    \item 收集少量成功/失败状态的标注样本
    \item 训练分类器 $C(s) \in [0, 1]$
    \item 用分类器的输出作为奖励
\end{itemize}

\subsection{LLM/VLM 作为奖励函数}

\begin{knowledgebox}{利用基础模型提供奖励}
最近的工作探索使用预训练的视觉语言模型（VLM）来自动生成奖励：
\begin{itemize}
    \item 用自然语言描述任务目标
    \item VLM 评估当前状态是否匹配语言描述
    \item 匹配程度作为奖励信号
\end{itemize}
这种方法利用了大规模预训练模型的世界知识，但奖励的精度可能不足以支撑精细的操作学习。
\end{knowledgebox}

\subsection{本章小结}

自动奖励获取的核心思路是用更自然的方式（图像、语言、分类器）替代人工编写的奖励函数。

\section{无重置学习}

\subsection{Reset-Free RL 的问题}

\begin{warningbox}{环境重置假设的不现实性}
标准 RL 假设每个 episode 结束后环境会被重置到初始状态。但在真实机器人场景中：
\begin{itemize}
    \item 物体被推倒后不会自己回到原位
    \item 机器人操作后环境状态会持续改变
    \item 让人类来重置环境代价极高且不可扩展
\end{itemize}
\end{warningbox}

\subsection{前向--后向控制器}

\begin{importantbox}{Forward-Backward Controller}
一种解决重置问题的方法是同时学习两个策略：
\begin{itemize}
    \item \textbf{前向策略} $\pi_f$：完成任务（从初始状态到目标状态）
    \item \textbf{后向策略} $\pi_b$：重置环境（从目标状态/任意状态回到初始状态）
\end{itemize}
训练时交替使用两个策略：前向策略执行任务，后向策略负责将环境状态恢复。
\end{importantbox}

\subsection{持续学习（Continual Learning）视角}

另一种视角是将 reset-free RL 视为\textbf{单一持续 episode}中的学习。机器人在不断变化的环境中持续学习，不依赖 episode 边界。这与持续学习（continual learning）的思想相吻合。

\subsection{本章小结}

无重置学习通过前向--后向控制器或持续学习范式来解决真实世界中的环境重置问题。

\section{自主目标设定}

\subsection{Goal-Conditioned RL 中的自主目标}

结合前面讲到的 goal-conditioned RL，自主目标设定的思路是让机器人\textbf{自己决定要学什么}：
\begin{enumerate}
    \item 维护一个"目标库"，记录已知的可达目标
    \item 选择适当难度的目标进行练习（不太简单也不太困难）
    \item 逐步扩展目标库，学习越来越复杂的技能
\end{enumerate}

\begin{importantbox}{课程学习（Curriculum Learning）}
自主目标设定本质上是一种自动课程学习：
\begin{itemize}
    \item 从简单目标开始（接近当前能力边界）
    \item 随着能力提升，逐步挑战更困难的目标
    \item 目标难度应处于"学习区"——既不太简单（无法学到新东西）也不太困难（总是失败）
\end{itemize}
\end{importantbox}

\subsection{本章小结}

自主目标设定让机器人能够自动构建学习课程，从易到难逐步扩展能力。

\section{安全与约束}

\subsection{安全探索}

自主 RL 中的安全性是一个关键问题：
\begin{itemize}
    \item 定义安全约束（如关节力矩限制、碰撞避免）
    \item 使用约束优化方法（Constrained MDP）确保策略满足安全约束
    \item 在不确定性高时采取保守行为
\end{itemize}

\subsection{本章小结}

安全约束是自主 RL 部署到真实世界的必要条件。

\section{总结与延伸}

自主 RL 的长期愿景是让机器人像人类一样在真实世界中自主学习：

\begin{enumerate}
    \item 用图像/语言/VLM 替代人工奖励设计
    \item 用前向--后向控制器解决无重置问题
    \item 用自主课程学习让机器人自己决定学什么
    \item 安全约束确保自主探索不会造成损害
    \item 这些组件的集成是实现真正自主机器人的关键路径
\end{enumerate}
