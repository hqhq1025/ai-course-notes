\section{导言：最后一讲的主旨}

这不是一节新的算法课，而是一场面向课程收尾的总览与展望。整场 talk 的核心问题很清楚：我们为什么要做视觉 AI，AI 应该只模仿人类的能力，还是应该超越、补足、乃至约束人类能力？

\begin{importantbox}{三条主线}
\begin{enumerate}
    \item \textbf{See what humans see}：机器先学会做人类已经很擅长的视觉理解。
    \item \textbf{See what humans don't see}：机器去看人类看不到、看不全、或看错的东西。
    \item \textbf{See what humans want to see}：机器和人类目标对齐，帮助人做事而不是简单替代人。
\end{enumerate}
\end{importantbox}

这三条主线对应了视觉 AI 的历史、技术演进和伦理边界。前两者是能力问题，第三个是价值问题。

\section{See What Humans See}

\subsection{为什么是视觉}

视觉一直是 AI 最自然的起点之一。人类在对象识别、快速场景理解、复杂背景下的目标检测上表现极强，这也让计算机视觉成为最早被认真挑战的 AI 子领域之一。

讲者回顾了视觉的进化背景：从动物最早获得光感能力，到视觉系统推动“进化军备竞赛”，再到人类依赖视觉完成生存、工作、学习与社交。视觉不是一个普通输入通道，而是很多智能行为的基础设施。

\begin{knowledgebox}{人类视觉的三个特点}
\begin{itemize}
    \item \textbf{速度快}：在极短时间内完成粗粒度到中粒度的识别。
    \item \textbf{对象中心}：人类天然擅长把场景拆成对象、部件和关系。
    \item \textbf{鲁棒}：即使光照、遮挡、角度变化，仍能维持较强识别能力。
\end{itemize}
\end{knowledgebox}

\subsection{从部件模型到统计学习}

早期视觉研究深受心理学启发，试图把对象理解为部件的组合。这个方向数学上优雅，但实际效果有限，因为真实世界里的外观变化太大，部件组合无法覆盖所有变化。

随后，计算机视觉进入统计学习阶段。这个阶段最关键的变化不是某个单独模型，而是范式转变：

\begin{itemize}
    \item 从手工规则转向参数学习。
    \item 从少量样本转向更多数据。
    \item 从单一模型转向一系列可比较的基准。
\end{itemize}

这也是机器学习真正进入视觉问题的开始。SVM、Bayes nets、random fields 等方法都在这一时期发挥作用，但它们仍然没有从根本上解决“泛化数据太少”的问题。

\subsection{ImageNet 和深度学习拐点}

ImageNet 项目是这个领域最重要的转折之一。讲者把它视为对心理学家提出的人类类别规模的回应：如果人类儿童能识别成千上万的视觉类别，那么机器也应该朝这个量级去构建数据集和模型。

ImageNet 的意义有三层：

\begin{enumerate}
    \item 数据规模首次逼近真实视觉世界的复杂度。
    \item 评测基准把不同算法放到可比较的轨道上。
    \item 2012 年 AlexNet 将数据、算力和卷积网络真正拧在一起，触发深度学习浪潮。
\end{enumerate}

\begin{importantbox}{2012 的三要素}
ImageNet 数据 + 卷积神经网络 + GPU 训练 = 现代深度学习在视觉上的爆发点。
\end{importantbox}

\subsection{从对象到关系}

视觉理解并不止于“这是什么”。同样重要的是“这些东西之间是什么关系”。讲者用两个例子说明了这一点：同样是 llama 和 person，关系不同，场景语义就完全不同。

这推动了 scene graph、Visual Genome 和关系理解等方向的发展。对象是节点，关系是边，属性是附加信息。视觉系统从“命名对象”推进到“描述场景”。

\begin{knowledgebox}{关系理解的价值}
\begin{itemize}
    \item 它让视觉系统从分类走向结构化理解。
    \item 它能帮助零样本或长尾关系泛化。
    \item 它为后续的图像描述、视频理解和多模态系统打基础。
\end{itemize}
\end{knowledgebox}

\subsection{从图像字幕到动态场景}

接下来的一步是把静态场景变成语言。图像 captioning、dense captioning、storytelling 等任务，都是“视觉 + 语言”结合的早期尝试。LSTM 和 CNN 的组合在这里发挥了重要作用。

但动态场景更难。对象在运动，行为在变化，摄像机本身也在移动。多对象、多主体活动理解要求模型不仅知道“有什么”，还要知道“谁在做什么、谁影响了谁、下一步可能是什么”。

\begin{warningbox}{动态场景的难点}
动态场景中的难点不只是检测准确率，而是关系、时序和行为的联合建模。很多问题到今天仍然没有完全解决。
\end{warningbox}

\subsection{总结}

这一部分的结论很明确：视觉 AI 的关键突破从来不是单一技巧，而是数据、算力和模型的共同收敛；而视觉理解的终点也不只是分类，而是对象、关系、语言和行为的统一理解。

\section{See What Humans Don't See}

\subsection{超越人类的识别能力}

这部分讨论的是：哪些视觉任务不是人类最擅长的，但机器可以做得更好。最典型的是 fine-grained classification，比如鸟类、车型、子类层级的识别。

这类任务的特点是：

\begin{itemize}
    \item 类别层级深。
    \item 外观差异细微。
    \item 数据难以覆盖长尾分布。
    \item 人类自身也不一定能稳定区分。
\end{itemize}

讲者提到的鸟类识别和车型识别，正是机器“看得更细”的代表。机器不是在模仿人，而是在成为一种超人类感知工具。

\subsection{视觉作为研究社会的工具}

更进一步，计算机视觉还可以用来研究人类社会本身。通过城市街景中的车型分布，可以推断教育水平、收入、投票倾向、环境差异等社会特征。

这类研究说明，AI 不只是回答“图里有什么”，还可以成为观察社会结构的显微镜。

\begin{importantbox}{AI 的新用途}
AI 可以从单张图像里提取人类肉眼不可能手工汇总的大规模统计规律，这使它成为社会科学和医学中的新型测量工具。
\end{importantbox}

\subsection{人类视觉的局限}

讲者用 Stroop test、change blindness 等经典心理学实验说明，人类视觉并不完美。我们读得很快，但注意力有限；我们能识别变化，但并不总能注意到变化。

这些局限在普通生活里只是有趣，但在医疗、工业和安全场景里就会变成风险。

\begin{warningbox}{注意力和偏差会带来真实代价}
人类不是无误差传感器。注意力有限、疲劳、经验偏差、社会偏差都会影响判断，而这些偏差在高风险场景中会放大成实际伤害。
\end{warningbox}

\subsection{医疗、偏见与隐私}

讲者重点举了医疗场景。手卫生、手术器械、ICU 监测、老人居家照护，这些都是典型的“暗空间”，也就是人手不足、注意力不足、长期高风险但又必须持续监控的地方。

AI 在这里的角色不是炫技，而是辅助：

\begin{itemize}
    \item 识别是否洗手。
    \item 监测 ICU 患者的移动状态。
    \item 在老人居家环境中进行早期异常检测。
    \item 在保护隐私的前提下提取必要信息。
\end{itemize}

与此同时，讲者也强调 bias 和 privacy 是必须正视的问题。视觉系统会继承训练数据和社会经验里的偏差，而隐私保护不是附加功能，而是设计目标本身。

\begin{knowledgebox}{隐私不是“后处理”}
如果应用场景要求“看见行为、但不要看见身份”，那就必须从传感器、表示学习到输出机制整体设计，而不是简单地把图像模糊一下。
\end{knowledgebox}

\subsection{总结}

这一部分的核心不是“AI 比人强”，而是“AI 能补人类的盲点”。在 fine-grained recognition、医疗监测和隐私保护场景里，视觉 AI 的价值在于把人类看不到、看不全、看不稳的部分补上。

\section{See What Humans Want to See}

\subsection{从替代到增强}

最后一部分是整个 talk 最重要的价值判断：AI 不应只被理解为替代劳动的工具，而应该首先被设计成增强人的工具。

讲者先谈了 labor anxiety。技术变迁总会重塑劳动力市场，但真正的问题不是“有没有变化”，而是“变化由谁承担，收益由谁获得”。在医疗、护理、老人照护等领域，人力短缺本来就是真问题，因此 AI 的增益尤其重要。

\begin{importantbox}{从 replace 到 augment}
AI 在很多场景下不是为了替代人，而是为了补足人力短缺、缓解疲劳、扩大可监测范围，并让人把精力放在更需要判断和关怀的地方。
\end{importantbox}

\subsection{Ambient Intelligence in Healthcare}

讲者给出的最系统案例是医疗中的 ambient intelligence。这里的思路不是把人完全取代，而是通过智能传感器和机器学习系统，持续从病房、ICU、家庭等环境中提取健康相关信号。

典型应用包括：

\begin{itemize}
    \item 手卫生监测。
    \item ICU 患者活动识别。
    \item 老人居家安全与行为跟踪。
    \item 感染、睡眠、饮食等健康模式识别。
\end{itemize}

这些系统共同指向一个目标：把稀缺的人力资源放到最需要人类判断的地方。

\subsection{Embodied AI 与机器人}

最后，讲者把视野推进到 embodied AI，也就是机器人。这里的关键不再只是“看见”，而是“看见以后做什么”。

机器人研究的难点在于：

\begin{itemize}
    \item 真实世界任务复杂。
    \item 数据采集和 benchmark 不够统一。
    \item 场景变化远大于实验室设定。
    \item 需要将感知、语言、规划和动作闭环起来。
\end{itemize}

讲者介绍了一个思路：用 LLM 生成任务分解，用 VLM 理解环境，再把结果转成 motion planning map，驱动机械臂执行。这个框架的意义不是“机器人终于会做一个固定动作”，而是让机器人更接近开放世界里的通用操作能力。

\begin{knowledgebox}{看到和做到的闭环}
视觉 AI 的终极目标之一，是把 perception 和 action 连起来。只有能理解环境并据此行动，AI 才真正成为具身智能。
\end{knowledgebox}

\subsection{Benchmark 的意义}

讲者最后提到的 benchmark，不只是评测准确率，更是评测现实性、生态有效性和任务复杂度。真正有价值的 benchmark 应该逼近真实世界，而不是只停留在实验室里“看起来很漂亮”的数据集。

\subsection{总结}

这一部分的结论是：AI 应该帮助人完成人的目标。它可以是增强工具、辅助工具、保护隐私的工具、也可以是把 perception 和 action 接起来的工具，但它首先应该尊重人类价值。

\section{全讲总结}

这节课把整个 CS231N 课程从技术层面和价值层面都收了回来。

\begin{importantbox}{三条最终 takeaway}
\begin{enumerate}
    \item \textbf{数据、算力和模型的收敛} 触发了现代视觉 AI 的爆发。
    \item \textbf{AI 的价值不止是复制人类能力}，还包括补足盲区、提高一致性和帮助人类理解世界。
    \item \textbf{最重要的是 human perspective}：AI 既可能放大偏差，也可能放大善意，关键在于我们如何设计它。
\end{enumerate}
\end{importantbox}

从对象识别到 scene graph，从 captioning 到动态场景，从医疗监测到机器人，视觉 AI 已经不再只是“看图分类”。它正在成为一种理解世界、辅助判断、支持行动、并且影响社会结构的基础能力。

\begin{itemize}
    \item 对研究者来说，它意味着继续推进更强的表示、更多样的数据和更真实的 benchmark。
    \item 对工程实践来说，它意味着把 AI 放进真实场景，处理偏差、隐私和可靠性。
    \item 对人类社会来说，它意味着要把技术进步和人的价值一起考虑。
\end{itemize}

\begin{knowledgebox}{最后一句话}
我们不仅要 build AI to see things，我们还要 build AI to help people.
\end{knowledgebox}
