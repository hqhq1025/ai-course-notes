\section{从单任务模型到 Foundation Models}

这是一节 guest lecture，主讲人是 Ranjay Krishna。他先回顾了课程前半段的标准范式：为每个任务单独收集数据、训练专用模型、在测试集上评估。随后他把视角切换到 foundation models，强调今天的核心变化是从``一个任务一个模型''转向``一个大模型适配很多任务''。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-002.jpg}
\caption{课程早期范式：每个任务单独训练一个专用模型\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第2页。}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-003.jpg}
\caption{Foundation models 的基本思路：先在大规模多样数据上预训练，再迁移到多个下游任务\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第3页。}

\subsection{什么是 Foundation Model}

课程里给出的判断标准很务实：foundation model 不是一个严格定义，而是一组典型特征。

\begin{importantbox}{Foundation model 的常见特征}
\begin{itemize}
    \item 对很多不同任务都足够\textbf{通用}、\textbf{鲁棒}
    \item 往往有\textbf{很多参数}
    \item 往往用\textbf{大量数据}训练
    \item 往往采用某种\textbf{自监督}预训练目标
\end{itemize}
\end{importantbox}

Krishna 也明确说明了本节课的范围：语言模型本身不展开讲，重点放在视觉和多模态 foundation models。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-005.jpg}
\caption{foundation models 的谱系：语言、分类、多模态、生成和 chaining\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第5页。}

\subsection{为什么视觉里也需要 foundation models}

视觉任务也面临同样的问题：图像分类、检测、分割、检索、问答，各自都有不同的数据和目标。把视觉模型做成 foundation model 的目标，不是只在一个 benchmark 上刷分，而是让同一套表示能迁移到尽量多的任务。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-021.jpg}
\caption{先预训练一个不依赖监督标签的编码器，再迁移到多个下游视觉任务\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第21页。}

下游任务可以很传统，比如图像分类、目标检测、语义分割；也可以更开放，比如检索、零样本识别、生成式 caption。

\section{CLIP：把图像和文本放进同一个空间}

这节课的核心案例是 CLIP。它把图像和文本都编码到同一个向量空间里，然后用对比学习把配对的图文拉近、把不匹配的样本推远。

\subsection{对比学习的直觉}

对比学习的逻辑并不复杂：如果两张图是同一个对象的不同增强版本，就应该在向量空间里很接近；如果是不同概念，就应该分开。把这个思路从``图像增强''推广到``图像-文本配对''，就是 CLIP 的关键。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-024.jpg}
\caption{CLIP 的基本结构：图像编码器和文本编码器各自输出一个向量，再用相似度做匹配\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第24页。}

\begin{importantbox}{CLIP 的训练目标}
CLIP 的训练数据是一大批图文对。训练时同时优化两个方向：
\begin{itemize}
    \item 图像向量应该最接近自己的文本描述
    \item 文本向量也应该最接近自己的图像
\end{itemize}
这是一种对称的对比损失，本质上就是在 batch 内做图文配对的拉近与拉远。
\end{importantbox}

\subsection{零样本分类怎么做}

CLIP 最重要的能力，不是单纯学到了一个更好的特征，而是能直接把文本当作分类器。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-026.jpg}
\caption{为每个类别用文本编码器生成一个向量，作为类别原型\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第26页。}

对新数据集，做法是：
\begin{enumerate}
    \item 为每个类别写一个文本提示词，比如 ``a photo of a dog''
    \item 用文本编码器得到每个类别的向量
    \item 用图像编码器得到输入图像的向量
    \item 用最近邻或点积相似度选出最匹配的类别
\end{enumerate}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-028.jpg}
\caption{把 CLIP 看成一个 1-NN 分类器：类别原型来自文本向量\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第28页。}

\begin{knowledgebox}{为什么 prompt 很重要}
CLIP 训练时见到的是句子片段，不是孤立词。于是 ``dog'' 往往不如 ``a photo of a dog'' 好用。对同一类别用多种 prompt 取平均，通常能再提升一点性能。
\end{knowledgebox}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-030.jpg}
\caption{同一类别用多种 prompt，再取平均向量作为最终类别表示\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第30页。}

\subsection{为什么 CLIP 会这么强}

Krishna 的回答很直接：scale。

\begin{itemize}
    \item 参数规模远大于传统监督分类器
    \item 数据规模来自互联网图文对，远超 ImageNet
    \item 训练目标允许大规模对比学习，易于扩展
\end{itemize}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-039.jpg}
\caption{CLIP 强的原因之一：模型和数据都大幅扩展\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第39页。}

这也解释了一个看起来反直觉的结果：没有人工标签的 CLIP，零样本性能可以接近甚至匹配一些强监督 ResNet。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-040.jpg}
\caption{CLIP 参数量远大于传统 ImageNet ResNet\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第40页。}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-041.jpg}
\caption{CLIP 的训练数据规模也远大于传统监督分类\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第41页。}

\subsection{CLIP 的优势}

\begin{importantbox}{CLIP-style 模型的优点}
\begin{enumerate}
    \item 点积相似度非常高效，训练和推理都快
    \item 零样本泛化强，能做 open-vocabulary 分类
    \item 容易和别的模型组合，适合 chaining
\end{enumerate}
\end{importantbox}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-045.jpg}
\caption{CLIP-style 模型在 ImageNet 上的表现已经足以作为新一代分类 foundation model\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第45页。}

\section{CoCa：加上生成目标}

CLIP 很强，但它本质上还是一个对比编码器。Krishna 接着介绍 CoCa，把 captioning decoder 加进来，做成 contrastive + captioning 的混合模型。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-042.jpg}
\caption{CoCa 在 CLIP 上加入生成式 caption loss\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第42页。}

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-043.jpg}
\caption{CoCa 的结构：编码器负责对比学习，decoder 负责生成 caption\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第43页。}

\begin{knowledgebox}{为什么要加 captioning}
对比学习擅长对齐图文，但生成目标会迫使模型保留更多细粒度语义。CoCa 的思路是：既要对齐，也要会说。
\end{knowledgebox}

\section{局限与失败模式}

这节课没有把 CLIP 神化。相反，后半部分花了不少时间讲它的局限。

\subsection{批量大小依赖}

CLIP 很依赖 batch size 来学到更细的概念。batch 越大，负样本越多，模型越容易分辨细粒度差异。但这个办法有上限，因为再大的 batch 也很难覆盖真正困难的组合关系。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-046.jpg}
\caption{CLIP-style 模型的优势和局限：高效、零样本，但也会被某些语义混淆击中\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第46页。}

典型例子是 compositionality：``there is a mug in some grass'' 和 ``there is some grass in a mug'' 词都一样，但语义完全不同，CLIP 却容易混淆。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-047.jpg}
\caption{CLIP 会混淆语序和组合关系，说明它对 compositionality 仍然不够强\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第47页。}

\subsection{hard negative fine-tuning}

一个自然的修补办法是 hard negative fine-tuning，用更难的反例逼模型学会更细粒度的区别。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-049.jpg}
\caption{用 hard negatives 继续微调可以缓解一些混淆问题\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第49页。}

但这也引入新的问题：如果负样本太硬，模型开始把本来应该接近的样本也强行推开，甚至制造出新的误差模式。讲义里把这类样本称为 ``hard positives'' 的陷阱，意思是说关系本来是复杂的，不一定能靠简单的正负对立讲清楚。

\begin{figure}[H]
\centering
\includegraphics[width=0.95\textwidth]{slides-images/slide-051.jpg}
\caption{过度依赖 hard negatives 也会制造新的训练偏差\protect\footnotemark}
\end{figure}
\footnotetext{Slides 第51页。}

\section{Chaining 和系统化使用}

CLIP 的另一个重要价值是它可以被别的模型调用，而不只是单独当分类器。Krishna 反复强调 chaining：把 foundation model 作为模块，再接上其他模型或 verifier。

\subsection{和其他模型组合}

一个常见模式是：CLIP 做视觉检索或候选生成，语言模型负责重写、总结或推理，最后再用别的模块做过滤和验证。这样系统能力会比单个模型更强。

\begin{itemize}
    \item 用 CLIP 找候选图像
    \item 用语言模型生成描述或解释
    \item 用 verifier 检查输出是否可信
\end{itemize}

\subsection{为什么要 verifier}

在 Q\&A 环节里，Krishna 提到一个现实观点：大模型输出不能总是直接暴露给用户。工业系统通常会先经过别的 verifier 或规则系统，降低 hallucination 风险。

\begin{warningbox}{单模型输出不一定可信}
foundation model 越强，系统越需要知道什么时候该信、什么时候不该信。把输出交给后处理模块验证，是实际产品系统里常见的做法。
\end{warningbox}

\subsection{会不会自己造工具}

他也回答了一个更前沿的问题：如果模型缺少某种能力，系统能不能自动收集训练数据、生成工具、再把能力补上。答案是理论上可以，实际还很早期，但方向明确。

\section{本章小结}

这节课的主线很清楚：

\begin{importantbox}{Lecture 16 的核心 takeaway}
\begin{enumerate}
    \item foundation model 的本质是大规模、通用、可迁移的预训练表示
    \item CLIP 用图文对比学习把图像和文本放进统一空间
    \item zero-shot 分类可以直接把文本当成类别原型
    \item scale 对 CLIP 的成功至关重要，尤其是数据和参数
    \item CoCa 说明在对比学习之外，加上生成目标也很有价值
    \item CLIP 不是万能的，尤其在 compositionality 和 hard negatives 上有明显局限
    \item 真正强的系统往往是 chaining 和 verifier，而不是单模型裸奔
\end{enumerate}
\end{importantbox}
