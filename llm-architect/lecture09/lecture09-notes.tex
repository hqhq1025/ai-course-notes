\section{引言：阅读模型厂 Tech Report 的主线}

K2.5 是月之暗面 (Moonshot) 发布的重要模型，题为 ``Visual Agency Intelligence''。本期完整串讲这篇庞杂的技术报告。

\begin{importantbox}{阅读模型厂技术报告的三条主线}
\begin{enumerate}
  \item \textbf{Data}：数据从哪来？如何合成？不同 domain 的 data 代表不同 task，对应不同 capability
  \item \textbf{Algorithm}：新算法是什么？不同阶段如何训练？
  \item \textbf{Training Infra}：如何优化 infra？如何做到 bug-free？跑得对、快、准
\end{enumerate}
\end{importantbox}

\section{Data：数据创造与能力构建}

\subsection{新 Domain = 新 Capability}

每种数据 domain 代表一个训练 task，对应一种能力。新模型比上一代多了新的 data domain，就多了新的 capability。

\subsection{合成数据方法}

K2.5 大量使用合成数据。对于新的 domain：
\begin{enumerate}
  \item 定义该 domain 的 benchmark
  \item 设计数据合成 pipeline
  \item 做 baseline training 验证有效性
\end{enumerate}

\subsection{本章小结}

数据是能力的基础。每个新增的数据 domain 都对应着模型的一项新能力。合成数据的质量和多样性决定了模型的上限。

\section{Algorithm：核心算法创新}

\subsection{General Reward Model (GRM)}

\begin{importantbox}{GRM 用于 Open-Ended RL Training}
传统 RL 训练中，reward 通常由规则定义（如数学题正确性）。但对于 open-ended 任务（如写作、对话），需要一个通用的 reward model 来评判输出质量。K2.5 深入探索了 GRM 在 RL post-training 中的应用。
\end{importantbox}

\subsection{多阶段训练流程}

K2.5 的训练流程包括：
\begin{enumerate}
  \item \textbf{Pre-training}：大规模多模态预训练
  \item \textbf{SFT}：监督微调，使用多种任务的高质量数据
  \item \textbf{RL Post-training}：使用 GRM 进行强化学习后训练
\end{enumerate}

\subsection{本章小结}

K2.5 的核心创新在 Data + Algorithm 的结合上。新数据带来新能力，新算法（特别是 GRM + RL）提升了模型在 open-ended 任务上的表现。

\section{原生多模态架构}

\subsection{从 KiMi VL 到 K2.5}

KiMi VL 是 K2.5 的前身，仍然分了纯语言模型和 VL 模型。到 K2.5，模型不再区分这两者，实现了\textbf{原生多模态}——一个模型同时具备语言理解和视觉理解能力。

\subsection{Visual Agency Intelligence}

K2.5 的核心定位不仅是一个多模态理解模型，更是一个具备\textbf{视觉 Agent 能力}的模型——它能够基于视觉输入做出决策、执行工具调用、完成复杂任务。

\subsection{本章小结}

K2.5 代表了从``视觉理解''到``视觉代理''的跨越，原生多模态架构是实现这一目标的基础。

\section{Agent Swarm：多 Agent 协作}

\subsection{概念}

Agent Swarm 是 K2.5 探索的一个方向：多个 Agent 实例协作完成复杂任务。每个 Agent 可以专注于不同的子任务，通过协调机制实现整体目标。

\subsection{与 Interleaved Thinking 的结合}

K2.5 的 Agent 能力与之前介绍的 Interleaved Thinking 密切相关：
\begin{itemize}
  \item 每步 think-act-observe 构成一个 Agent 循环
  \item 可以串行执行 200--300 步工具调用
  \item 多个 Agent 可以并行处理不同子任务
\end{itemize}

\subsection{本章小结}

Agent Swarm 是 K2.5 的重要创新方向，将单 Agent 的 Agentic 能力扩展到多 Agent 协作场景。

\section{Training Infra}

\subsection{关键挑战}

\begin{itemize}
  \item \textbf{Bug-free}：大规模训练中的 bug 代价极高
  \item \textbf{跑得快}：训练效率直接影响研发速度
  \item \textbf{跑得准}：数值精度、梯度稳定性
\end{itemize}

\subsection{RL Training 的 Infra 挑战}

Agentic RL Training 的 infra 挑战尤为突出：
\begin{itemize}
  \item 需要在线 rollout（模型推理 + 环境交互）
  \item 多步工具调用增加了复杂度
  \item 异步 training + inference 的协调
\end{itemize}

\subsection{本章小结}

Training Infra 是模型厂竞争力的重要组成部分。高效、稳定、可扩展的 infra 是做好大规模 RL training 的前提。

\section{总结与延伸}

\begin{enumerate}
  \item 阅读模型厂 Tech Report 的三条主线：Data, Algorithm, Infra
  \item K2.5 实现了原生多模态，不再区分纯语言和 VLM
  \item GRM + RL 是 open-ended 任务训练的关键
  \item Agent Swarm 将 Agentic 能力扩展到多 Agent 协作
  \item Training Infra 的重要性不亚于算法创新
\end{enumerate}
