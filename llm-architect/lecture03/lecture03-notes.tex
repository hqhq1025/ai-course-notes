\section{引言：从 Long CoT 到 Interleaved Thinking}

自 2024 年 9 月 OpenAI 推出 o1 (Reasoning Model) 以来，以及 2025 年 1 月 DeepSeek R1 的发布，``Thinking Model'' 成为大模型领域的重要范式。然而，传统的 Long Chain-of-Thought (Long CoT) 存在明显不足。

K2 Thinking 提出了一种全新的范式------\textbf{Interleaved Thinking}（交错推理/交错思考），面向 Agentic 场景，通过强化学习训练模型将思考与回答交替进行。

\begin{importantbox}{传统 Long CoT 的问题}
\begin{itemize}
  \item 在最终 answer 之前有一个漫长的 long-CoT 过程
  \item 首字延迟（Time to First Token, TTFT）极高，用户体验差
  \item 难以定义细粒度的 reward，过程监督困难
\end{itemize}
\end{importantbox}

\section{Interleaved Thinking 的核心理念}

\subsection{概念起源}

Interleaved Reasoning 最早由苹果在 2025 年的一篇工作中提出：``交错推理 for 大语言模型通过强化学习''。核心思想是将思考和回答分解成多步，每步先思考（think）再回答（answer），交替进行。

\subsection{多步分解的优势}

以一个多跳推理问题为例：``柏林墙倒塌之后第五年的奥斯卡最佳影片导演是谁？''

\textbf{传统 Long CoT}：一次性长推理，容易在中间环节出错，最终得到错误答案。

\textbf{Interleaved Thinking}：
\begin{enumerate}
  \item Think 1：召回柏林墙倒塌年份 $\to$ 1989
  \item Answer 1：第五年 = 1994
  \item Think 2：检索 1994 年奥斯卡最佳影片 $\to$ 《阿甘正传》
  \item Answer 2：导演是 Robert Zemeckis
\end{enumerate}

\begin{importantbox}{Interleaved Thinking 的三大优势}
\begin{itemize}
  \item \textbf{首字延迟低}：每步 think-answer 很短，可能只有几个 token
  \item \textbf{细粒度 reward}：可以对每步的输出定义正确性 reward，方便过程监督
  \item \textbf{Agentic 友好}：不需要用户参与，模型自行执行多轮 tool call
\end{itemize}
\end{importantbox}

\subsection{面向 Agentic 场景}

K2 Thinking 可以执行串行的 200--300 步工具调用，无需人类参与。从 message list 角度看，它是 user-assistant 的单轮交互，但 assistant 内部完成了大量的 think-act-observe 循环。

\begin{knowledgebox}{Interleaved vs. Long CoT 的 message list 对比}
\begin{itemize}
  \item \textbf{Long CoT}：$[\text{user}, \underbrace{\text{<think>}\ldots\text{</think>}}_{\text{很长的推理链}}, \text{answer}]$
  \item \textbf{Interleaved}：$[\text{user}, \text{think}_1, \text{act}_1, \text{obs}_1, \text{think}_2, \text{act}_2, \text{obs}_2, \ldots, \text{answer}]$
\end{itemize}
\end{knowledgebox}

\subsection{RL 训练方式}

Interleaved Thinking 的步数不是预先设定的，而是通过 RL 训练\textbf{自动涌现}的。模型学会根据问题复杂度动态决定需要多少步 think-answer 循环。

\subsection{本章小结}

Interleaved Thinking 是对 Long CoT 范式的重要改进，将推理链打散为多个 think-answer 步骤。它降低了首字延迟、支持细粒度 reward、天然适配 Agentic 工具调用场景。

\section{K2 Thinking 的技术特点}

\subsection{自我验证能力}

模型在每一步 answer 后可以自我验证（self-verification），判断当前结果是否正确，决定是否需要继续推理或回溯。

\subsection{与 General Reward Model 的结合}

在 RL 训练中，可以为每步 think-answer 定义独立的 reward signal，而不仅仅是最终答案的 outcome reward。这使得训练信号更加密集和稳定。

\subsection{本章小结}

K2 Thinking 的技术核心在于通过 RL 训练模型的 interleaved reasoning 能力，结合细粒度 reward 和自我验证机制。

\section{总结与延伸}

\begin{enumerate}
  \item Interleaved Thinking 是继 Long CoT 之后 Thinking Model 的重要范式演进
  \item 核心优势：低延迟、细粒度 reward、Agentic 友好
  \item 步数由 RL 训练自动涌现，模型动态决定推理深度
  \item K2 可执行 200--300 步串行工具调用，无需人类介入
\end{enumerate}
