\section{引言}

本讲由 CS25 课程讲师联合主讲，涵盖两大主题：涌现能力与中间引导推理（由 Seonghee 讲授）和 AI 智能体（由 Div 讲授）。

\section{涌现能力（Emergent Abilities）}

\subsection{定义与现象}

\begin{importantbox}{涌现能力的定义}
涌现能力是指在大模型中出现、但在小模型中不存在的能力，且无法通过小模型性能的外推来预测。表现为``相变''（phase transition）：性能在某个临界规模突然跃升。
\end{importantbox}

Wei et al. (2022) 发现多种任务（模运算、单词重组、多步推理等）在训练 FLOPs 达到约 $10^{22}$--$10^{23}$ 时出现性能跃迁。

\subsection{涌现的争议}

\begin{warningbox}{涌现可能是评估指标的假象}
Stanford 2023 年的研究指出，涌现现象可能是评估指标非线性（如精确匹配）的产物：如果使用连续评估指标，性能提升看起来是渐进的而非突变的。这引发了关于``涌现到底是模型的属性还是指标的属性''的深入讨论。
\end{warningbox}

\subsection{本章小结}
涌现能力的存在性仍有争议，但规模化确实带来了质的变化。

\section{中间引导推理}

\subsection{思维链提示（Chain-of-Thought）}

\begin{importantbox}{思维链的核心思想}
让模型在给出最终答案前先生成中间推理步骤。通过在 few-shot 示例中加入推理过程，模型学会``展示思考过程''，显著提升数学和逻辑推理能力。CoT 本身也是一种涌现能力——仅在大模型中有效。
\end{importantbox}

\subsection{BabyLM Challenge}

\begin{knowledgebox}{用少量数据训练语言模型}
BabyLM Challenge 探索一个有趣的问题：人类儿童在约 1 亿词的语言输入下就能习得语言能力，而 LLM 需要数万亿 token。能否用类似人类学习量（$\sim$100M 词）的数据训练出高效的语言模型？
\end{knowledgebox}

\subsection{本章小结}
中间推理引导和小数据效率是提升 LLM 推理能力的两个互补方向。

\section{AI 智能体（Agents）}

\subsection{智能体的定义}

将 LLM 从``回答问题的工具''升级为``在环境中行动的系统''。

\begin{importantbox}{神经计算单元类比}
可以把 LLM 类比为``神经 CPU''——它是核心处理单元，但完整的智能体还需要：
\begin{itemize}
\item \textbf{记忆}：短期（上下文窗口）和长期（向量数据库）
\item \textbf{规划}：任务分解和策略选择
\item \textbf{工具}：搜索引擎、计算器、代码执行器
\item \textbf{行动}：在数字或物理环境中执行操作
\end{itemize}
\end{importantbox}

\subsection{计算机交互智能体}

讲者展示了一个能控制计算机的智能体 demo：通过自然语言指令，智能体可以操作网页浏览器、文件系统等。这相当于一个``通用 API''——不需要为每个应用编写接口，一个智能体就能完成所有操作。

\subsection{安全与可靠性}

\begin{warningbox}{智能体的安全挑战}
\begin{itemize}
\item \textbf{错误纠正}：智能体难以自动检测和修复推理错误
\item \textbf{用户权限}：需要分层权限系统（如禁止访问银行账户，但允许访问购物网站）
\item \textbf{沙箱隔离}：确保智能体不会误删重要文件或在高风险场景中造成不可逆损害
\item \textbf{不可逆操作}：金融交易、数据删除等操作需要额外确认机制
\end{itemize}
\end{warningbox}

\subsection{本章小结}
AI 智能体是 LLM 落地的关键范式，但安全性和可靠性仍需大量基础设施建设。

\section{总结与延伸}

本讲覆盖了当前 LLM 研究的两大前沿：涌现能力（及其争议）和 AI 智能体。涌现与规模化的关系仍在研究中；智能体从``对话''走向``行动''，但可靠性是实际部署的最大障碍。
