\section{引言：机制可解释性}

Chris Olah 来自 Anthropic，是机制可解释性（Mechanistic Interpretability）领域的先驱。他的研究目标是\textbf{理解神经网络内部的算法}——不只是观察输入输出行为，而是弄清楚参数如何映射到具体的计算过程。

\begin{importantbox}{什么是机制可解释性？}
机制可解释性试图\textbf{逆向工程}神经网络：
\begin{itemize}
    \item 不是``模型为什么做出这个预测''（归因/解释）
    \item 而是``模型内部实际执行了什么算法''（机制理解）
    \item 类似于逆向工程一个编译后的程序——从二进制代码恢复源代码逻辑
\end{itemize}
\end{importantbox}

\section{注意力头的两个独立电路}

\subsection{QK 电路与 OV 电路}

\begin{importantbox}{注意力头的数学分解}
一个注意力头可以分解为两个\textbf{完全独立}的电路：
\begin{itemize}
    \item \textbf{QK 电路}（Query-Key）：决定``哪些 token 应该关注哪些 token''
    $$A = \text{softmax}\left(\frac{(XW_Q)(XW_K)^T}{\sqrt{d_k}}\right)$$
    \item \textbf{OV 电路}（Output-Value）：决定``当一个 token 被关注时，什么信息被传递''
    $$\text{Output} = A \cdot (XW_V W_O)$$
\end{itemize}
这两个电路是\textbf{独立的参数矩阵}，尽管公式看起来它们是纠缠的。
\end{importantbox}

\begin{knowledgebox}{直觉理解}
想象一个图书馆系统：
\begin{itemize}
    \item QK 电路 = 目录/索引系统——决定去哪本书找信息
    \item OV 电路 = 书的内容——决定找到后获取什么信息
\end{itemize}
你可以改变索引系统而不改变书的内容，反之亦然。
\end{knowledgebox}

\subsection{本章小结}

注意力头的 QK 和 OV 电路的独立性是理解 Transformer 内部计算的基础。

\section{归纳头（Induction Heads）}

\subsection{什么是归纳头？}

\begin{importantbox}{归纳头——In-Context Learning 的机制}
归纳头是一种在 Transformer 中自发涌现的\textbf{两层注意力模式}，执行以下算法：

\textbf{规则}：如果序列中出现过``A B ... A''模式，则预测下一个 token 为``B''。

\textbf{机制}：
\begin{enumerate}
    \item \textbf{第一层}：一个``前一个 token''注意力头将 B 的信息复制到 B 之后的位置
    \item \textbf{第二层}：归纳头从当前 A 回头寻找之前 A 的下一个位置（现在包含 B 的信息），并将 B 复制到输出
\end{enumerate}
这本质上是\textbf{上下文最近邻}——在序列上下文中查找匹配模式并复制答案。
\end{importantbox}

\subsection{归纳头与 In-Context Learning}

Chris Olah 提出了一个重要假说：\textbf{归纳头可能是 in-context learning 的核心机制}。

\begin{itemize}
    \item 归纳头在训练过程中突然出现（相变），伴随着 loss 的急剧下降
    \item 出现时机与模型获得 in-context learning 能力的时机一致
    \item 在不同规模的模型中都观察到这一现象
\end{itemize}

\begin{warningbox}{归纳头不是全部}
虽然归纳头是最显著的 in-context learning 机制，但可能还有其他贡献因素。Chris 强调这是当前的``工作假说''，而非最终结论。
\end{warningbox}

\subsection{本章小结}

归纳头是一种优雅的两层注意力模式，可能是 Transformer in-context learning 能力的核心机制。

\section{信息流与残差流}

\subsection{残差流视角}

\begin{knowledgebox}{残差流（Residual Stream）}
将 Transformer 视为一条``残差流''——信息沿着残差连接流动，每个注意力头和 MLP 层\textbf{向流中读写信息}：
\begin{itemize}
    \item 残差流是模型各层之间的``通信总线''
    \item 早期层可以将信息写入残差流，供后续层读取
    \item 这解释了为什么跨层的注意力模式组合能产生复杂行为
\end{itemize}
\end{knowledgebox}

\subsection{组合性}

Transformer 的强大来自\textbf{组合性}——多个注意力头的效果可以组合，产生比单个头复杂得多的行为。通过残差流视角，我们可以分析信息如何在层间流动和组合。

\subsection{本章小结}

残差流提供了分析 Transformer 信息流动的统一框架，使得跨层行为的理解成为可能。

\section{总结与延伸}

Chris Olah 的工作展示了机制可解释性的巨大潜力——我们可以打开 Transformer 的``黑箱''，发现其中的具体算法。归纳头的发现是这一研究方向的标志性成果，它不仅提供了对 in-context learning 的机制理解，也暗示了更多复杂电路等待发现。
