\section{引言：后 ChatGPT 时代的对齐故事}

Nathan Lambert 是 Allen Institute for AI（AI2）的研究科学家，专注于 RLHF 和开放语言模型对齐。本讲以 ChatGPT 发布为分水岭，系统回顾了开源社区在对齐领域的快速进展。

\begin{knowledgebox}{从 Claude Shannon 到 ChatGPT}
语言模型的思想可追溯到 Claude Shannon 的字符级序列预测。Anthropic 将其模型命名为 Claude 正是致敬这一渊源。从 Shannon 到 GPT-4，核心思想始终是自回归损失函数。
\end{knowledgebox}

\section{自回归语言模型基础}

\subsection{自回归损失函数}

$$\mathcal{L} = -\sum_{t=1}^{T} \log P(x_t | x_{<t}; \theta)$$

给定训练样本 ``the cat sat on''，模型学习预测每个位置的下一个 token。这个简单的损失函数在足够大的数据和模型上产生了惊人的能力。

\subsection{预训练 $\to$ 微调 $\to$ 对齐}

\begin{importantbox}{现代 LLM 训练三阶段}
\begin{enumerate}
\item \textbf{预训练}：在互联网文本上学习通用语言表示
\item \textbf{监督微调}（SFT）：在指令-响应对上学习遵循指令
\item \textbf{对齐}（RLHF/DPO）：根据人类偏好进一步优化
\end{enumerate}
\end{importantbox}

\subsection{本章小结}
简单的自回归目标是一切的基础，后续的 SFT 和对齐都是在此基础上的精细化调整。

\section{ChatGPT 时代的开源爆发}

\subsection{时间线}

\begin{itemize}
\item 2022.11：ChatGPT 发布
\item 2023.02：LLaMA 开源，开启开源 LLM 时代
\item 2023.03：Alpaca（Stanford）、Vicuna 等开源对齐模型涌现
\item 2023.04：Open Assistant 众包数据集
\item 2023.07：LLaMA 2 + RLHF
\item 2024.04：LLaMA 3，接近 GPT-4 水平
\end{itemize}

\begin{knowledgebox}{开源追赶闭源}
Google 内部泄露的备忘录预测：``我们没有护城河，OpenAI 也没有。''开源社区通过数据工程和对齐技术快速缩小了与闭源模型的差距。LLaMA 3 70B Instruct 的性能已接近 GPT-4。
\end{knowledgebox}

\subsection{本章小结}
ChatGPT 之后，开源社区在不到两年内实现了惊人的追赶。

\section{RLHF 的实践}

\subsection{奖励模型训练}

\begin{importantbox}{奖励模型是对齐的核心}
奖励模型学习预测人类偏好：给定 prompt 和两个回答，预测哪个更好。训练数据来自人类标注员的偏好对比。奖励模型的质量直接决定 RLHF 的上限。
\end{importantbox}

\subsection{PPO 训练的挑战}

\begin{warningbox}{PPO 训练不稳定}
PPO（Proximal Policy Optimization）在 RLHF 中的应用面临多个挑战：
\begin{itemize}
\item 四个模型需要同时在内存中（策略、参考策略、奖励模型、价值网络）
\item 超参数敏感——KL 系数、学习率需要精心调节
\item 奖励黑客（Reward Hacking）——模型学会``欺骗''奖励模型而非真正改善质量
\item 训练成本高，需要大量 GPU 资源
\end{itemize}
\end{warningbox}

\subsection{DPO 及其变体}

DPO（Direct Preference Optimization）通过重新参数化消除了显式奖励模型的需要：

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]$$

\begin{knowledgebox}{DPO 的优势与局限}
\textbf{优势}：训练简单、稳定、内存占用小。\\
\textbf{局限}：在某些评估上不如 PPO；可能过拟合偏好数据中的表面模式。\\
实际中，DPO 和 PPO 各有适用场景，社区仍在探索最佳实践。
\end{knowledgebox}

\subsection{本章小结}
RLHF 有效但复杂，DPO 提供了更简便的替代方案。

\section{评估的困境}

\subsection{评估不是已解决的问题}

\begin{warningbox}{对齐评估缺乏可靠信号}
预训练阶段有 MMLU、HellaSwag 等相对稳定的基准；但对齐评估缺乏类似的``黄金标准''。MT-Bench 和 AlpacaEval 有用但有偏差。Open LLM Leaderboard 成功的原因恰恰是因为``有一点信号总比没有好''，而非评估本身很可靠。
\end{warningbox}

\subsection{Chatbot Arena}

LMSYS 的 Chatbot Arena 通过众包 ELO 评分提供了最接近真实用户偏好的信号，但样本量和人群偏差仍是问题。

\subsection{本章小结}
评估是对齐研究的最大瓶颈，目前没有完美方案。

\section{合成数据与未来方向}

\begin{itemize}
\item \textbf{合成数据生成}：使用 LLM 生成更多训练 token，缓解数据稀缺
\item \textbf{Self-play}：模型与自身对弈进行训练（如 OpenAI 的超级对齐方向）
\item \textbf{过程奖励}：不仅奖励最终答案，还奖励推理过程中的每一步
\end{itemize}

\section{总结与延伸}

本讲从历史视角全面审视了语言模型对齐的现状。核心教训：(1) 简单的自回归目标 + 规模化是基础；(2) 对齐技术（SFT/RLHF/DPO）是将原始能力转化为有用行为的桥梁；(3) 评估仍然是最大痛点；(4) 合成数据可能是下一个突破口。
