\section{引言：音频领域的 Transformer}

Prateek Verma 来自 Stanford，介绍了 Transformer 在音频、语音和音乐领域的应用。音频处理与文本/图像有本质差异：音频信号是\textbf{高采样率的时间序列}（如 16kHz 语音、48kHz 音乐），这对 Transformer 的序列长度提出了极大挑战。

\begin{knowledgebox}{音频表示的特殊性}
\begin{itemize}
    \item \textbf{原始波形}：每秒数万个采样点，序列极长
    \item \textbf{频谱图/梅尔频谱}：时频表示，类似于图像，但时间轴有因果性
    \item \textbf{离散音频 token}：通过 VQ-VAE 等方法将音频离散化为 token 序列
\end{itemize}
\end{knowledgebox}

\section{从不同视角理解 Transformer}

Prateek 提供了 Transformer 自注意力的一个有趣视角：

\begin{importantbox}{自注意力 vs. 小波变换}
传统信号处理使用\textbf{固定的}基函数（如小波、傅里叶基）来分析信号。Transformer 的自注意力可以看作一种\textbf{自适应的}基函数——它根据输入数据动态决定哪些时间点/频率点需要更多关注。这使得 Transformer 在处理非平稳信号时天然优于固定基方法。
\end{importantbox}

\subsection{本章小结}

Transformer 通过自适应注意力机制，在音频领域提供了超越传统信号处理方法的灵活性。

\section{采样级别的音频语言建模}

\subsection{SampleRNN 到 Transformer}

早期的音频生成模型（如 WaveNet、SampleRNN）逐采样点自回归生成，速度极慢。Transformer 可以在以下层面进行``语言建模''：

\begin{itemize}
    \item \textbf{采样级别}：直接预测下一个音频采样值（精度最高但最慢）
    \item \textbf{帧级别}：预测频谱帧（平衡精度与效率）
    \item \textbf{Token 级别}：先将音频编码为离散 token，再做语言建模（最高效）
\end{itemize}

\subsection{上下文长度的挑战}

\begin{warningbox}{音频的上下文长度瓶颈}
以 16kHz 采样率的 100ms 音频为例，就有 1600 个采样点。标准注意力需要 $1600^2 \approx 2.56 \times 10^6$ 的内存。对于几秒的音频片段，这个问题更加严峻。解决方案包括：
\begin{itemize}
    \item 稀疏注意力模式
    \item 线性注意力近似
    \item 分层处理（先局部再全局）
    \item 音频 tokenization 降低序列长度
\end{itemize}
\end{warningbox}

\subsection{本章小结}

音频的高采样率对 Transformer 的上下文长度提出了独特挑战，催生了多种高效注意力方案。

\section{语音和音乐的 Transformer 应用}

\subsection{语音识别与合成}

\begin{itemize}
    \item \textbf{ASR（自动语音识别）}：Transformer 已成为主流架构（如 Whisper）
    \item \textbf{TTS（文本到语音）}：Tacotron 2 + Transformer、VITS 等
    \item \textbf{语音翻译}：端到端的跨语言语音转换
\end{itemize}

\subsection{音乐生成与理解}

\begin{itemize}
    \item \textbf{Music Transformer}（Huang et al.）：基于相对注意力的音乐生成
    \item \textbf{Jukebox}（OpenAI）：使用 VQ-VAE + Transformer 的原始音频音乐生成
    \item \textbf{音乐信息检索}：节拍检测、乐器识别、情感分析
\end{itemize}

\begin{importantbox}{研究中的诚实性}
Prateek 引用 DeepMind 研究者的观点，强调在发表论文时应诚实报告训练技巧的贡献——很多时候性能提升来自于工程技巧而非模型创新。区分``模型实际贡献''和``训练技巧贡献''对于领域的健康发展至关重要。
\end{importantbox}

\subsection{本章小结}

Transformer 在语音和音乐领域的应用正在快速发展，从识别到生成全面覆盖。

\section{总结与延伸}

音频领域的 Transformer 应用展示了这一架构的通用性，同时也揭示了其在处理长序列信号时的独特挑战。随着高效注意力机制和音频 tokenization 方法的发展，Transformer 在音频领域的能力将持续提升。
